단축키 가이드 팝업 기능의 디자인 시안 요청서

📅 2026-03-26 👤 From: PM Kyle → To: Desi 📋 디자인 항목 6건 🔖 Priority: Phase 1
🎯 디자인 목표: 사용자가 ? 키 또는 헤더 아이콘을 통해 모든 단축키를 즉시 확인할 수 있는 모달 팝업. GitHub, Linear, Gmail의 패턴을 참고하되, Testably의 디자인 시스템에 맞게 제작.

1 기능 개요 & 컨텍스트

현재 상태

  • Testably에는 50+개 키보드 단축키가 구현되어 있으나, 사용자에게 노출되는 곳이 Focus Mode의 kbd 힌트뿐입니다.
  • ? 키 핸들러가 이미 코드에 존재하지만 (useKeyboardShortcuts.ts Line 61), 연결된 UI가 없습니다.
  • TestRail, Qase, Zephyr 등 QA 도구 중 인앱 단축키 가이드를 제공하는 제품이 없어, Testably가 선도할 수 있습니다.

레퍼런스 제품

GitHub (Primary Reference)

? → 모달 오버레이. 페이지별 컨텍스트 단축키 표시. 단일키 비활성화 옵션.

Linear (Secondary Reference)

? + 사이드바 메뉴. 모달 + 검색 기능. G-chord 네비게이션 (우리와 동일 패턴).

Gmail

? → 카테고리별(Inbox/Conversation/Compose). 새 창 열기 옵션. ON/OFF 토글.

Figma

하단 패널 (비모달). 검색 가능. 작업 중 열어둘 수 있음. 키 입력 시 하이라이트.

5 Design Item 4 — Focus Mode 내 가이드 접근

Focus Mode는 전체 화면 오버레이이므로 헤더 버튼이 보이지 않습니다. 별도 접근 방법이 필요합니다.

6 Design Item 5 — Interaction Flows

6-1. 열기 플로우

트리거애니메이션초기 상태
? 키overlay fade-in(150ms) + modal slide-up(150ms)검색 필드에 포커스, "Current Page" 섹션 펼침
헤더 ⌨ 클릭동일동일

6-2. 검색 인터랙션

  • 타이핑 즉시 필터링 (debounce 없이 실시간)
  • 매칭: 키 이름 또는 액션 이름에서 검색 (예: "pass" → P — Pass & Next 행 표시)
  • 매칭 텍스트에 볼드 하이라이트
  • 매칭 없는 카테고리는 통째로 숨김
  • 모든 텍스트 삭제 시 전체 목록 복원
  • 빈 결과: 중앙에 "No shortcuts match your search" + ri-search-line 아이콘 48px, color: #cbd5e1

6-3. 닫기 플로우

방법애니메이션
Esc 키overlay fade-out(100ms) + modal slide-down(100ms)
✕ 버튼 클릭동일
Overlay 클릭동일

6-4. 토글 인터랙션

  • "Single-key shortcuts" 토글 OFF → 수정자 없는 단축키 행에 opacity: 0.4 + strikethrough 적용
  • 토글 상태는 localStorage에 저장 (세션 유지)
  • 토글 OFF 시 실제 단축키도 비활성화됨 (코드 연동 필요)

기존 디자인 시스템과의 일관성

  • 모달 패턴: 기존 QuickCreateTC, AI Assist 모달과 동일한 overlay + 중앙 정렬 + Esc 닫기
  • 검색 패턴: Command Palette의 검색 UI 참조 (동일한 input 스타일)
  • kbd 스타일: Focus Mode의 기존 kbd 힌트 스타일 재사용 (monospace, light bg)
  • 컬러: 기존 STATUS_CONFIG 컬러 토큰 활용 (Pass: green, Fail: red, Block: amber, Retest: violet)
  • 아이콘: Remix Icons 사용 (ri-keyboard-line, ri-search-line, ri-close-line)
