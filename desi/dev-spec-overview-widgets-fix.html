A-2. 기획안 의도 (pm-plan-project-overview-redesign.html §5-2)

기획안에서는 Q6 "긴급 조치할 것은?"에 직접 답변하는 위젯으로, 4가지 트리거 조건을 정의했습니다:

#트리거 조건아이콘/색상의미현재 구현
T-13회+ 연속 실패 TC (Repeat Failure)🔴 ri-close-circle-fill #EF4444같은 TC가 3번 이상 연속 failed → Flaky 또는 근본 결함없음
T-2Blocked 3일+ TC🟡 ri-forbid-line #F59E0Bblocked 상태로 3일 이상 방치 → 환경/의존성 이슈 고착없음
T-3Milestone 기한 ≤10일 + Pass Rate <70%🔴 ri-alarm-warning-fill #EF4444마감 임박한데 품질 미달 → 릴리스 위험없음
T-4미실행 7일+ TC (Untested Aging)⚪ ri-time-line #94A3B87일 넘게 untested → 커버리지 사각지대없음

A-3. 갭 요약

4개 트리거 중 0개 구현됨. 현재 로직은 기획안에 정의된 트리거 조건과 완전히 다릅니다.
현재 = "Run에 failed > 0이면 표시" (단순 필터)
기획 = "TC 레벨에서 반복 실패, 장기 blocked, 마감 위험, 미실행 방치를 감지" (다차원 분석)

A-4. 수정 스펙

T-1: 3회+ 연속 실패 TC 감지

항목스펙
쿼리 로직test_results 테이블에서 각 TC의 최근 3건 결과를 가져와, 3건 모두 status = 'failed'인 TC를 추출
SQL 핵심 
-- 각 TC의 최근 3건 결과를 가져와 모두 failed인 TC
WITH ranked AS (
  SELECT test_case_id, status,
    ROW_NUMBER() OVER (
      PARTITION BY test_case_id ORDER BY created_at DESC
    ) AS rn
  FROM test_results
  WHERE run_id IN (
    SELECT id FROM test_runs WHERE project_id = :projectId
  )
)
SELECT test_case_id
FROM ranked
WHERE rn <= 3
GROUP BY test_case_id
HAVING COUNT(*) = 3
  AND COUNT(*) FILTER (WHERE status = 'failed') = 3
표시 형태🔴 {TC 이름} — "3회 연속 실패" + 최근 실패 시간
링크클릭 → 해당 TC의 Run Detail 페이지
우선순위P0 가장 높은 심각도로 정렬

T-2: Blocked 3일+ TC 감지

항목스펙
쿼리 로직test_results에서 status = 'blocked'이고 created_at이 3일 이상 전인 TC (이후 상태 변경 없는 것)
구현 방식 
// 각 TC의 최신 결과가 blocked이고 3일+ 경과
const blockedTCs = latestResultPerTC
  .filter(r => r.status === 'blocked')
  .filter(r => {
    const blockedDate = new Date(r.created_at);
    const diffDays = (Date.now() - blockedDate.getTime())
                     / (1000 * 60 * 60 * 24);
    return diffDays >= 3;
  });
표시 형태🟡 {TC 이름} — "Blocked {N}일째" + 소속 Run 이름
우선순위P0

T-3: Milestone 기한 ≤10일 + Pass Rate <70%

항목스펙
쿼리 로직milestones 테이블에서 end_date가 오늘부터 10일 이내인 항목 필터 → 해당 Milestone에 연결된 Run들의 종합 Pass Rate 계산 → 70% 미만이면 경고
구현 방식 
// 기한 임박 + 품질 미달 Milestone
const atRiskMilestones = milestones
  .filter(m => m.status !== 'completed')
  .filter(m => {
    if (!m.end_date) return false;
    const daysLeft = Math.ceil(
      (new Date(m.end_date) - Date.now()) / 86400000
    );
    return daysLeft >= 0 && daysLeft <= 10;
  })
  .filter(m => {
    const msRuns = testRuns.filter(r => r.milestone_id === m.id);
    const totalPassed = msRuns.reduce((s, r) => s + (r.passed||0), 0);
    const totalAll = msRuns.reduce((s, r) =>
      s + (r.passed||0) + (r.failed||0) + (r.blocked||0)
      + (r.retest||0) + (r.untested||0), 0);
    const passRate = totalAll > 0 ? (totalPassed / totalAll) * 100 : 0;
    return passRate < 70;
  });
표시 형태🔴 {Milestone 이름} — "기한 {N}일 남음, Pass Rate {X}%"
링크클릭 → Milestone Detail 페이지
우선순위P0

T-4: 미실행 7일+ TC (Untested Aging)

항목스펙
쿼리 로직Active Run에 포함된 TC 중, test_results에 결과가 없거나 최신 결과가 untested이면서 해당 Run의 created_at이 7일 이상 전인 TC 수
구현 방식 
// 7일+ 미실행 TC 카운트
const untestedAging = activeRuns
  .filter(r => {
    const runAge = (Date.now() - new Date(r.created_at)) / 86400000;
    return runAge >= 7;
  })
  .reduce((count, run) => {
    const untestedInRun = (run.test_case_ids || []).filter(tcId =>
      !latestResultMap.has(tcId) ||
      latestResultMap.get(tcId) === 'untested'
    );
    return count + untestedInRun.length;
  }, 0);
표시 형태⚪ {N}개 TC — "7일+ 미실행" + 관련 Run 수
우선순위P1 가장 낮은 심각도

B-3. P1 수정 스펙 — 날짜 그루핑

현재 플랫 리스트를 날짜 그룹으로 분리합니다:

// 날짜 그루핑 로직
function groupByDate(events) {
  const today = new Date();
  today.setHours(0, 0, 0, 0);
  const yesterday = new Date(today);
  yesterday.setDate(yesterday.getDate() - 1);

  const groups = { today: [], yesterday: [], earlier: [] };

  events.forEach(event => {
    const eventDate = new Date(event.created_at);
    eventDate.setHours(0, 0, 0, 0);

    if (eventDate.getTime() === today.getTime())
      groups.today.push(event);
    else if (eventDate.getTime() === yesterday.getTime())
      groups.yesterday.push(event);
    else
      groups.earlier.push(event);
  });

  return groups;
}

UI: 그룹 헤더는 text-[0.6875rem] font-semibold text-gray-400 uppercase tracking-wider py-1.5로, "Today", "Yesterday", "Earlier this week" 텍스트. 구분선은 border-t border-gray-100 mt-2.
