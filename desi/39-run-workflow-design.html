§5. Keyboard Shortcuts Implementation
5.1 Hook Design
// useRunWorkflowShortcuts.ts export function useRunWorkflowShortcuts({ onPass, onFail, onBlock, onRetest, onAddResult, onPrev, onNext, enabled = true, // disable when modal/popup open }: ShortcutHandlers) { useEffect(() => { if (!enabled) return; const handler = (e: KeyboardEvent) => { // Skip if user is typing in an input/textarea if (e.target instanceof HTMLInputElement || e.target instanceof HTMLTextAreaElement) return; switch(e.key.toLowerCase()) { case 'p': onPass(); break; case 'f': onFail(); break; case 'b': onBlock(); break; case 'r': onRetest(); break; case 'a': onAddResult(); break; case 'k': case 'arrowup': e.preventDefault(); onPrev(); break; case 'j': case 'arrowdown': e.preventDefault(); onNext(); break; } }; window.addEventListener('keydown', handler); return () => window.removeEventListener('keydown', handler); }, [enabled, onPass, onFail, onBlock, onRetest, onAddResult, onPrev, onNext]); }
5.2 Hint Display
Keyboard hints are shown below each button as [P] [F] [B] [R] [A]. Font: 0.5625rem (9px), color rgba(255,255,255,0.7) for filled buttons, inherit with opacity 0.5 for outline buttons. Nav buttons show K/J hints as absolute-positioned labels below the button (font-size 0.5rem, color #94A3B8).
5.3 Context-Aware Disabling
Shortcuts are disabled when: (1) InlineNotePopup is open, (2) AddResultModal is open, (3) Focus Mode is active (Focus Mode has its own shortcut set defined in file 38). Inside popup/modal, only Ctrl+Enter (submit) and Esc (close) are active.
§6. Auto-Advance & "Add & Next"
6.1 localStorage Persistence
// Key: 'testably:autoAdvance' // Value: 'true' | 'false' // Default: false (unchecked) const [autoAdvance, setAutoAdvance] = useState(() => { return localStorage.getItem('testably:autoAdvance') === 'true'; }); const toggleAutoAdvance = (value: boolean) => { setAutoAdvance(value); localStorage.setItem('testably:autoAdvance', String(value)); };
6.2 Auto-Advance Timing
  • Quick Status (Pass/Retest): Save to DB → 500ms delay → navigate to next TC
  • Inline Popup (Save & Next / Skip & Next): Save to DB → close popup → 500ms delay → navigate
  • "Add & Next" button in modal: Save to DB → close modal → 500ms delay → navigate
  • 500ms delay provides visual feedback (brief status indicator flash on the row)
  • If current TC is the last in the run, show toast: "All test cases completed" instead of advancing

§10. Jira Integration
10.1 Feature Overview
When a TC is marked Failed or Blocked, users can: (1) search and link an existing Jira issue, or (2) create a new Jira issue with TC data auto-filled. Available in both the inline note popup (compact button row) and the full Add Result modal (dedicated section).

10.4 Jira Search Behavior
  • Debounced search: 300ms after last keystroke
  • Searches by issue key (exact match) and summary (fuzzy match)
  • Dropdown: max 5 results, shows icon (bug/task/story), key, summary, status
  • Click to select → adds to linked issues list
  • Multiple issues can be linked to one result
  • Linked issue chip: key (monospace, blue) + summary + remove (×) button

10.8 Data Model
-- Jira integration tables CREATE TABLE jira_connections ( id uuid PRIMARY KEY DEFAULT gen_random_uuid(), workspace_id uuid REFERENCES workspaces(id), jira_site_url text NOT NULL, -- e.g. "myteam.atlassian.net" access_token text, -- encrypted OAuth token refresh_token text, connected_by uuid REFERENCES users(id), created_at timestamptz DEFAULT now() ); CREATE TABLE result_jira_issues ( id uuid PRIMARY KEY DEFAULT gen_random_uuid(), result_id uuid REFERENCES results(id) ON DELETE CASCADE, jira_key text NOT NULL, -- e.g. "PROJ-142" jira_url text NOT NULL, summary text, status text, created_by uuid REFERENCES users(id), created_at timestamptz DEFAULT now() ); CREATE INDEX idx_result_jira_issues_result ON result_jira_issues(result_id);
Jira Connection Required: If no Jira connection exists for the workspace, show an inline banner: "Connect Jira to link issues → Go to Settings" instead of the search/create UI. The Jira OAuth setup lives in Project Settings → Integrations → Jira.
Escalation Context Passing: When "Create Jira Issue" is clicked from the inline popup, carry over: (1) the status, (2) any typed note, (3) the current TC context. If clicked from the Add Result modal, additionally carry: (4) step results, (5) attachments already added.
§9. Implementation Priority
Phase 1 — Core (P0)
  • ① Quick Status Bar — 4 buttons + Add Result + Nav. Pass/Retest instant-apply.
  • ② Inline Note Popup — Fail/Block popup with note, Skip & Next, Save & Next, escalation.
  • ④ Add Result Modal Refactor — Status-driven field visibility. Remove rich text toolbar. 3-button footer.
Phase 1 — Enhanced (P1)
  • ⑤ Keyboard Shortcuts — P/F/B/R/A/J/K with context-aware disabling.
  • ⑥ Auto-Advance — Checkbox + localStorage + 500ms delay + "Add & Next" button.
Phase 2 — Step-Level & Jira (P1–P2)
  • ③ Step-Level Inline Results — Mini Pass/Fail per step. Toggle bar progress. Data model (step_results table).
  • ⑨ Jira Integration — OAuth connection setup, search+link existing issues, create new issue with auto-fill. Inline popup buttons + modal section.
  • Step progress visualization in toggle bar.
  • 3-option auto-advance setting (Off / Next untested / Next in list).
Design Principles: (1) Escalation Path — Quick Status → Inline Popup → Full Modal → Focus Mode. (2) Status-Driven UI — Passed = minimal friction, Failed = expanded detail capture. (3) Keyboard-First — Every action reachable via keyboard. (4) Zero-Friction Pass — Single click/key to pass and advance. (5) Fail-to-Fix Pipeline — Fail → Note → Jira Issue in one flow, no context switching.
