배경 및 Context

CEO 피드백: "Focus Mode의 경우, test step만 표시되고 있어. Expected result도 함께 표시되어야 하고 첨부된 사진이 있다면 그것도 함께 표시되어야 해. 그리고 test case ID, folder, priority, tags, comments 등 실제 테스트를 수행할 때 필요한 모든 정보가 표시되어야 할 것 같은데."

PM 분석 결과: CEO 의견에 전적으로 동의. 경쟁사(TestRail/Qase/Zephyr) 대비 8개 핵심 기능 부족. 3-Tier 정보 구조로 확장하여 "집중"을 유지하면서 자급자족(Self-contained) 실행 환경으로 진화시킵니다.

3-Tier 정보 구조 — 설계 원칙

🎯 Tier 1 — 메인 콘텐츠

항상 표시. 테스트 실행에 직접 필요한 정보.

  • Test Steps + Expected Result
  • Precondition
  • Attachments / Screenshots
  • Status 버튼 + Note 입력
📋 Tier 2 — 메타데이터 바

헤더 아래 컴팩트 1줄. 한눈에 TC 맥락 파악.

  • TC-ID (TC-001 포맷)
  • Priority 뱃지
  • Folder
  • Tags
  • Assignee 아바타
💬 Tier 3 — 확장 패널

접이식. 클릭/단축키로 펼침.

  • Comments (이전 댓글)
  • Execution History (이전 결과)
핵심 원칙: Focus Mode의 가치는 "집중"입니다. 정보를 추가하되 복잡도를 높이면 안 됩니다. Tier 1은 항상 보이고, Tier 2는 한 줄로 압축, Tier 3은 필요할 때만 펼칩니다.

시나리오 1: 일반 테스트 실행 플로우

Step사용자 행동시스템 반응
1Focus Mode 진입 (Cmd+Shift+F)풀스크린 표시. 첫 번째 TC 로드. Metadata Bar에 TC-ID·Priority·Folder·Tags·Assignee 표시.
