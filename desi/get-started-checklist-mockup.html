Click Action Flows — 각 항목 클릭 시 동작
1
Create your first project
Click → /projects?action=create → Create Project 모달 자동 오픈 → markStep('createProject')
?action=create 파라미터 → ProjectsContent.tsx에서 useEffect로 모달 자동 트리거
2
Write your first test case
Click → /projects/{id}/testcases?action=create → Quick Create TC 모달 → markStep('createTestcase')
프로젝트 없으면 → Toast: "프로젝트를 먼저 만들어주세요" + Step 1 하이라이트
3
Try AI test generation
Click → /projects/{id}/testcases?action=ai-generate → AI Generate 모달 자동 오픈 → markStep('tryAi')
핵심 개선: 기존에는 TC 목록으로만 이동 → 개선 후 AI 모달이 즉시 오픈됨
4
Run your first test
Click → /projects/{id}/runs?action=create → Add Run 모달 자동 오픈 → markStep('runTest')
TC 없으면 → Toast: "테스트 케이스를 먼저 추가하세요" + Step 2 하이라이트
5
Invite a team member
Click → /settings?tab=members&action=invite → Invite 모달 자동 오픈 → markStep('inviteMember')
버그 수정: /settings#team → /settings?tab=members. Settings는 hash가 아닌 query param 사용.
6
Connect Jira Optional
Click → /settings?tab=integrations → markStep('connectJira')
버그 수정: /settings#integrations → /settings?tab=integrations. Jira 저장 성공 시 자동 완료.
Dev Note — Screen 4: Click Action Flows
핵심: ?action= 쿼리 파라미터 → 각 페이지의 useEffect에서 감지 → 모달 자동 오픈.   OnboardingChecklist.tsx의 linkTo 값 업데이트 필요 (6개 항목 모두).   markStep: OnboardingContext를 통해 하위 컴포넌트에서 접근.

Dev Note — Screen 5: Dependency Toasts
Soft Lock 방식: 항목은 항상 클릭 가능하되, 선행 조건 미충족 시 toast 표시.   Toast 종류: (A) amber = 선행조건 안내 + 해당 스텝 바로가기 링크, (B) green = 완료 축하.   체크리스트 내 인라인 힌트: 선행 미충족 항목에 amber dashed border + "Requires:" 서브텍스트.   Toast 자동 소멸: 4초 후 fade-out. 액션 링크 클릭 시 즉시 해당 스텝으로 이동.
