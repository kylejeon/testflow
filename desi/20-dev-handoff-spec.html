1. Overview & Design Philosophy

These mockups represent an ultra-subtle compact treatment applied to the 5 inner project pages. The goal is to increase information density without making the UI feel cramped. Changes are intentionally minimal: padding reductions of 10–15% and font size reductions of only 1–2px from production values.

The shared chrome (project header bar and tab bar) is intentionally unchanged from production. Only the inner content area below the tab bar receives the compact treatment.

Additionally, file 19 specifies a "Don't show again" checkbox for the existing Onboarding Welcome modal.

Key principle: If you hold the compact mockup next to the production page, the difference should be barely perceptible — just slightly tighter spacing. No layout structure changes, no color changes, no font weight changes.

12. Implementation Notes

Approach: Custom Tailwind Values

Since the compact values don't map to standard Tailwind classes (e.g., 1.375rem title, 1.3125rem card padding), use one of these approaches:

  1. Arbitrary values in Tailwind: p-[1.3125rem], text-[1.375rem]
  2. Extend tailwind.config.js with a compact theme extension
  3. CSS custom properties defined at :root level for easy toggling between standard and compact modes
Recommendation: Option 3 (CSS custom properties) gives the most flexibility if we later want to let users toggle compact mode via a preference setting. Define --content-pad, --card-pad, --title-size, etc. and swap values.

Responsive Considerations

  • The Folders panel (page 14) should collapse to a hamburger on viewports below 768px
  • Stat card grids should stack to 1-col on mobile (grid-cols-1 below 640px)
  • Run/milestone/session cards are already single-column and responsive

Accessibility

  • All interactive elements use <button> elements (not <div> with click handlers)
  • Checkboxes in table and onboarding use native <input type="checkbox"> with <label> association
  • Color is never the sole indicator: priority dots always pair with text labels, status badges use text
  • Focus states should use ring-2 ring-indigo-300 pattern consistently
