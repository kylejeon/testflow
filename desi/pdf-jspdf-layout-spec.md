# jsPDF Layout Spec — 8-Page Executive Report
## Dev2용 정확한 mm 좌표 레이아웃 스펙

> **이 문서는 디자인 목업 `dashboard-export-pdf-mockup.html`의 1:1 구현을 위한 좌표 참조 시트입니다.**
> **모든 값은 mm 단위이며, jsPDF API 호출에 직접 사용됩니다.**
> **추측 금지 — 여기 없는 값은 Desi에게 문의하세요.**

---

## 0. 글로벌 상수 & 공통 요소

### 0.1 페이지 규격
```
PAGE_W    = 210       // A4 가로
PAGE_H    = 297       // A4 세로
MARGIN    = 20        // 상하좌우 동일
CONTENT_X = 20        // 콘텐츠 시작 x
CONTENT_W = 170       // 210 - 20*2
CONTENT_Y = 20        // 일반 페이지 콘텐츠 시작 y (헤더 14mm + 여백 6mm)
CONTENT_H = 257       // 297 - 20*2
RIGHT_X   = 190       // 우측 정렬 기준 x (210 - 20)
CENTER_X  = 105       // 중앙 정렬 기준 x (210 / 2)
```

### 0.2 색상 팔레트 (jsPDF setTextColor / setFillColor / setDrawColor용)
```js
const C = {
  // 기능 색상
  PRIMARY:    [99, 102, 241],   // Indigo
  PASS:       [16, 163, 127],   // Green
  FAIL:       [239, 68, 68],    // Red
  BLOCK:      [249, 115, 22],   // Orange
  RETEST:     [234, 179, 8],    // Yellow
  UNTESTED:   [203, 213, 225],  // Gray

  // Priority 색상
  PRI_CRIT:   [239, 68, 68],    // Red
  PRI_HIGH:   [245, 158, 11],   // Amber
  PRI_MED:    [99, 102, 241],   // Indigo
  PRI_LOW:    [148, 163, 184],  // Gray

  // 텍스트
  DARK:       [15, 23, 42],     // 제목, 주요 수치
  LIGHT:      [100, 116, 139],  // 라벨, 캡션
  WHITE:      [255, 255, 255],

  // 배경 & 보더
  BG_LIGHT:   [248, 250, 252],  // 카드/테이블 헤더 배경
  BORDER:     [226, 232, 240],  // 구분선, 테이블 보더

  // 상태 배경 (연한 색)
  BG_GREEN:   [236, 253, 245],
  BG_RED:     [254, 242, 242],
  BG_AMBER:   [255, 251, 235],
  BG_INDIGO:  [238, 242, 255],
};
```

### 0.3 폰트 체계
```
NotoSansKR-Regular  → "normal"
NotoSansKR-Bold     → "bold"
helvetica fallback (NotoSansKR 미로드 시)

크기 용도:
  18pt  Bold    — P1 커버 프로젝트명
  14pt  Bold    — Readiness 신호 텍스트
  13pt  Bold    — 페이지 제목 (사용처: 없음, 헤더에 9pt 사용)
  16pt  Bold    — KPI 대형 값
  11pt  Bold    — 섹션 제목, Readiness Score
  10pt  Bold    — P1 헤더 로고, 마일스톤 카드 제목
   9pt  Normal  — 본문, 테이블 데이터, 정보값
   9pt  Bold    — 요약 바 값, Priority 라벨, Contribution %
   8pt  Bold    — 테이블 헤더, 요약 바 라벨
   8pt  Normal  — 마일스톤 상세, 푸터 텍스트, 날짜/보조
   7pt  Normal  — KPI 라벨, 차트 축 라벨, 범례, 카드 타겟
   7pt  Bold    — 뱃지 텍스트
```

### 0.4 공통 헤더 (P2~P8)
```
요소: Header Bar
  x: 0, y: 0, w: 210, h: 14
  배경: setFillColor(99,102,241) → rect(0, 0, 210, 14, 'F')

요소: Logo "Testably"
  x: 20, y: 8 (baseline)
  폰트: 10pt, bold, WHITE
  정렬: left
  코드: pdf.text("Testably", 20, 8)

요소: Page Title (예: "Quality Scorecard")
  x: 190, y: 8 (baseline)
  폰트: 9pt, normal, WHITE
  정렬: right
  코드: pdf.text(title, 190, 8, {align:'right'})
```

### 0.5 공통 푸터 (모든 페이지)
```
요소: Footer Divider
  x1: 20, y1: 285, x2: 190, y2: 285
  선: 0.3pt, BORDER
  코드: pdf.setLineWidth(0.3*0.3528) → pdf.line(20, 285, 190, 285)

요소: Project Name (좌)
  x: 20, y: 290 (baseline)
  폰트: 8pt, normal, LIGHT
  코드: pdf.text(projectName, 20, 290)

요소: "Generated by Testably — {date}" (중앙)
  x: 105, y: 290 (baseline)
  폰트: 8pt, normal, LIGHT
  정렬: center
  코드: pdf.text(genText, 105, 290, {align:'center'})

요소: "Page X of Y" (우)
  x: 190, y: 290 (baseline)
  폰트: 8pt, normal, LIGHT
  정렬: right
  코드: pdf.text(pageText, 190, 290, {align:'right'})
```

### 0.6 KPI 카드 공통
```
카드 크기: w=53, h=22
카드 간격: gap=5.5  (3열: x좌표 = 20, 78.5, 137)
모서리: radius=2
배경: BG_LIGHT, 보더: 0.5pt BORDER
내부 패딩: 4mm (모든 방향)

  라벨 텍스트:
    x: cardX+4, y: cardY+6 (baseline)
    폰트: 7pt, normal, LIGHT

  값 텍스트:
    x: cardX+4, y: cardY+13 (baseline)
    폰트: 16pt, bold, DARK

  델타 텍스트 (▲/▼):
    x: cardX+4, y: cardY+17 (baseline)
    폰트: 8pt, normal
    색상: ▲ 긍정=PASS, ▼ 부정=FAIL
    기호: "▲ " 또는 "▼ " 접두사 + 값

  타겟/보조:
    x: cardX+4, y: cardY+20 (baseline)
    폰트: 7pt, normal, LIGHT
```

### 0.7 테이블 공통
```
헤더 행:
  높이: 8mm
  배경: BG_LIGHT (rect fill)
  텍스트: 8pt, bold, DARK
  y baseline: rowY + 5.5 (행 높이 중앙)

데이터 행:
  높이: 7mm
  배경: 홀수=WHITE, 짝수=BG_LIGHT (zebra)
  텍스트: 9pt, normal, DARK
  y baseline: rowY + 5 (행 높이 중앙)

행 구분선:
  각 행 하단: 0.3pt, BORDER
  코드: pdf.line(tableX, rowY+rowH, tableX+tableW, rowY+rowH)

셀 패딩: 좌측 4mm (첫번째 셀 x = colX + 4)
```

### 0.8 섹션 제목
```
폰트: 11pt, bold, DARK
y: 해당 섹션 시작 y 좌표 (baseline)
x: 20 (CONTENT_X)
제목 아래 콘텐츠까지 간격: 5~7mm
```

### 2.1 Status Distribution Bar (y=20~38)
```

const HEADER_H = 8;

const tcPages = Math.ceil(tcs.length / TC_PER_PAGE);

for (let p = 0; p < tcPages; p++) {

  if (p === tcPages - 1) {

## 부록: jsPDF 유틸리티 함수 시그니처

```js
// 둥근 사각형
pdf.roundedRect(x, y, w, h, rx, ry, style)
// style: 'F'=fill, 'S'=stroke, 'FD'=fill+stroke

// 선
pdf.line(x1, y1, x2, y2)

// 원
pdf.circle(cx, cy, r, style)

// 텍스트
pdf.text(text, x, y, options)
// options: {align: 'left'|'center'|'right', baseline: 'top'|'middle'|'bottom'}
// 주의: y는 baseline 기준 (기본값)

// 대시 패턴
pdf.setLineDashPattern(dashArray, dashPhase)
// 실선 복귀: pdf.setLineDashPattern([], 0)

// 투명도
pdf.setGState(new pdf.GState({opacity: 0.6}))
// 복귀: pdf.setGState(new pdf.GState({opacity: 1.0}))

// pt → mm 변환 (lineWidth용)
// jsPDF setLineWidth는 현재 unit 기준
// mm 모드에서: pdf.setLineWidth(0.1) = 0.1mm ≈ 0.3pt
// 0.3pt = 0.3 * 0.3528 = 0.106mm
// 0.5pt = 0.5 * 0.3528 = 0.176mm
// 0.7pt = 0.7 * 0.3528 = 0.247mm
// 1.0pt = 1.0 * 0.3528 = 0.353mm
```
