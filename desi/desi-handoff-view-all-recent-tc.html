1 PM 기획 결정: URL 파라미터 방식

3가지 옵션 분석
옵션설명장점단점
A. 새 페이지 /recent-testcases 별도 라우트 독립적 디자인 자유 중복 코드, 라우트 관리 비용, 유사 UI 분산
B. URL 파라미터 ✅ /testcases?period=7d&sort=created_desc 기존 인프라 재사용, 일관된 UX, 최소 공수 TestCaseList에 sort 파라미터 확장 필요
C. 모달 Overview 페이지 위에 모달 컨텍스트 유지 TC가 많으면 모달 스크롤 문제, 검색·필터 제한
✅ PM 권장: Option B — 기존 /testcases 페이지 + URL 파라미터

"View All Failed TC"에서 이미 /testcases?status=failed&period=7d 패턴을 확립했습니다. 동일한 패턴을 적용하여 /testcases?period=7d&sort=created_desc로 구현합니다.

근거: ① "View All Failed TC"와 동일 패턴 → 사용자 학습 비용 제로. ② useFilteredTestCases hook이 이미 period=7d 필터를 지원. ③ TestCaseList.tsx에 sort URL 파라미터 확장만 추가하면 됨 (약 30분 Dev 공수). ④ 검색, 프로젝트 필터, Priority 필터 등 기존 기능 무료 획득.

2 동적 헤더 & "← Back to Overview"

URL 파라미터 sort=created_desc가 있고 period=7d일 때, 헤더가 "Recently Added Test Cases"로 변경됩니다. 이전 "View All Failed TC"가 status=failed일 때 "Failed Test Cases"로 변경되는 것과 동일 패턴입니다.

3 Period 드롭다운 (7d 기본 선택)

필터 영역

기존 TestCaseList의 필터 영역을 그대로 사용합니다. sort=created_desc로 진입 시 Period 드롭다운이 "Last 7 days"로 pre-selected 됩니다. Status 탭 바도 그대로 표시되며, 사용자가 자유롭게 추가 필터링 가능합니다.

[Filter Area — 기존 레이아웃 재사용] ┌─────────────────────────────────────────────────────────────────────┐ │ 🔍 [Search test cases… ] [All Projects ▼] [All Priority ▼] [Last 7 days ▼] │ ├─────────────────────────────────────────────────────────────────────┤ │ All(42) │ Failed(3) │ Passed(28) │ Blocked(1) │ Untested(10) │ └─────────────────────────────────────────────────────────────────────┘ ↑ Period 드롭다운이 "Last 7 days"로 pre-selected ↑ Status 탭은 "All"이 기본 활성 (View All Failed TC는 "Failed"가 기본)
차이점: "View All Failed TC" 진입 시 status=failed 탭이 활성화되지만, "View All Recently Added" 진입 시 status=all 탭이 활성화됩니다. "최근 추가된 TC"는 모든 상태를 포함하기 때문입니다.
Period 기본값'7d' (URL에서 읽음)
Status 기본값'all' (URL에 status 없으면 all)
Period 변경 시URL 업데이트 → 데이터 re-fetch → 테이블 리렌더
사용자가 Period를 변경하면subtitle 자동 변경 (예: "Last 30 days" → "…in the last 30 days")

4 테이블 컬럼 & 정렬

기존 TestCaseList 테이블을 확장합니다. "Recently Added" 맥락에서는 Created 컬럼이 추가되고, 기본 정렬이 Created 내림차순입니다 (최신순). Fail Count / Last Failed 컬럼은 숨기되, 사용자가 status=failed 탭 클릭 시 자동 표시됩니다.

컬럼 정의
ColumnWidth정렬 가능비고
☑ (체크박스)36px—Bulk action용
TC-ID100px✅ sort=tc_id프로젝트 prefix 포함 (예: AUTH-041)
Titleflex✅ sort=titletruncate with ellipsis
Project120px—프로젝트명 pill badge
Priority80px✅ sort=priorityCritical/High/Med/Low pill
Status90px—실행 상태 badge (Passed/Failed/…)
Assignee100px—Avatar + name (truncate)
Created NEW120px✅ sort=created_desc/asc상대 시간 (예: "2 hours ago") + tooltip 절대 시간
컬럼 가시성 규칙:
• sort=created_desc 진입 시: Created 컬럼 표시, Fail Count / Last Failed 숨김
• status=failed 진입 시: Fail Count / Last Failed 표시, Created 숨김 (기존 동작)
• 기본 진입: Created와 Fail Count 모두 숨김 (기존 동작 유지)
