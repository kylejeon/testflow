2.2 변경사항 요약

변경내용이유
추가 Description 필드 — textarea, optional Run의 목적/범위를 기록할 수 있어야 함. 팀원 간 커뮤니케이션에 필수.
추가 Assignees 필드 — 멀티 선택, 기본값: 본인 DB에 assignees[] 컬럼이 이미 있지만 현재 자동으로 본인만 할당됨. 팀원 할당 필요.
개선 Tags 입력 — comma-separated text → 칩 UI + 자동완성 현재 텍스트 입력은 실수 유발. 프로젝트 내 기존 태그 자동완성 + 칩 UI가 표준.
개선 Milestone + State를 한 줄로 모달 세로 길이 절약. 두 필드 모두 단순 드롭다운이므로 병렬 배치 가능.
개선 "Include all" 옵션에 Active TC 수 표시 사용자가 현재 몇 개 TC가 Run에 포함되는지 즉시 확인 가능.
제거 Configuration 필드 Configuration 관리 기능이 아직 없어 빈 드롭다운만 표시됨. Phase 2에서 추가.
제거 Issues 필드 Issues 관리 기능이 없어 빈 드롭다운 + 동작 안하는 "+ Add" 버튼. Phase 2에서 Jira 연동 후 추가.
제거 NOTES 탭 클릭해도 아무 반응 없음. 빈 탭은 사용자에게 "깨진 기능" 인상을 줌.
제거 Milestone "+" 버튼 동작 안함. Milestone은 프로젝트 설정에서 관리.
DB 영향: Description 필드 추가 시 test_runs 테이블에 description TEXT 컬럼 추가 필요. Assignees는 기존 assignees TEXT[] 컬럼 활용.

3. TC 선택 팝업 — 현재 분석

현재 TC 선택 팝업은 3컬럼 레이아웃입니다: 왼쪽 폴더 트리 + 중앙 TC 리스트 + 오른쪽 필터 패널.

3.1 현재 구현 기능

기능구현 상태문제점
폴더 트리 (왼쪽)✅ 동작함폴더별 선택 수/전체 수 표시. 잘 동작.
TC 리스트 (중앙)✅ 동작함체크박스 + TC명 + Priority 표시. 기본적.
검색 (중앙 상단)✅ 동작함title + description 검색. 문제 없음.
헤더 체크박스 (전체 선택)✅ 동작함현재 필터된 TC만 선택/해제.
Priority 필터✅ 동작함High/Medium/Low 체크박스. 기본 전체 선택.
Tags 필터✅ 동작함드롭다운 + 칩 표시. 잘 동작.
Select All / Deselect All✅ 동작함전체 TC 대상 (필터 무관). 혼란 소지 있음.
Footer 카운트✅ 동작함selected/total selected 표시.

3.2 누락/개선 필요 항목

항목현재개선안
Lifecycle Status 필터 ❌ 없음. Draft/Active/Deprecated 구분 없음. 추가 "Include Draft TCs" 토글 + Deprecated 완전 숨김 (이전 기획대로)
TC 행에 lifecycle 배지 ❌ 없음. Priority만 표시. 추가 TC명 옆에 Draft/Active 배지. Deprecated는 표시하지 않음.
폴더별 "전체 선택" 버튼 ❌ 없음. 폴더 클릭 시 필터만 됨. 추가 폴더 항목에 호버 시 체크박스 또는 "Select all" 아이콘
Execution Status 필터 ❌ 없음. 추가 Passed/Failed/Untested 등 실행 상태별 필터 (오른쪽 패널)
선택된 TC 요약 칩 ⚠️ 단순 숫자만 ("12/48 selected") 개선 상태별 분류: "32 Active, 5 Draft" 칩 표시
폴더별 선택 진행률 ⚠️ 숫자만 (3/12) 개선 미니 프로그레스 바 또는 체크 아이콘 추가
"Select All" 버튼 스코프 ⚠️ 전체 TC 대상 — 필터 무관하여 혼란 수정 현재 필터된 + 현재 폴더의 TC만 대상으로 변경
Draft TC 선택 시 경고 ❌ 없음. 추가 이전 기획(spec-tc-workflow-design-handoff.html §6)의 amber 경고 배너

8. 엣지케이스

엣지케이스동작
"Include all" 선택 + Deprecated TC 존재 "Include all"은 Active TC만 포함. "All 48 Active test cases will be included" 표시. Draft/Deprecated 제외.
"Include all" 선택 + Draft TC 존재 Draft TC는 제외. Active만 포함. 사용자가 Draft를 포함하려면 "Select specific" → 토글 ON 필요.
TC가 0개인 프로젝트 TC 선택 영역 대신 "아직 테스트 케이스가 없습니다. TC를 먼저 생성해주세요." + TC 생성 링크
모든 TC가 Deprecated "선택 가능한 TC가 없습니다." Empty state. Run 생성 불가.
Name 빈 상태로 제출 Name 입력 필드 빨간 border + "Run 이름을 입력하세요" 에러 메시지
"Select specific" 선택 + 0개 선택으로 제출 "최소 1개 이상의 TC를 선택하세요" 경고. 제출 차단.
Edit 모드에서 모달 열기 기존 Run 데이터로 폼 pre-fill. 버튼 "Update run". 기존 TC 선택 상태 복원.
Assignees에서 본인 제거 허용. 빈 assignees 배열도 가능. "팀원이 지정되지 않았습니다" 경고 없음.
