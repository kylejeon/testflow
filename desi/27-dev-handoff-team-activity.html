1 Heatmap Overview & Layout

The Team Activity heatmap follows the GitHub contribution graph pattern. It displays 52 weeks (1 year) of daily team activity, ending at the current date, inside a single card component.

PropertyValue
Grid dimensions52 columns (weeks) × 7 rows (Sun–Sat)
Total cells364 (up to 3 future cells hidden at end of current week)
OrientationRows = days of week (Sun at top, Sat at bottom). Columns = weeks (oldest at left, newest at right).
Date range52 weeks back from the current week's Saturday. Right edge = this week.
Container cardWhite card, border: 1px solid #E2E8F0, border-radius: 0.75rem
Card padding0.875rem 1rem (14px 16px)
Card margin-bottom1rem
The heatmap uses grid-template-columns: repeat(52, 1fr) with aspect-ratio: 1 on cells. This means cells auto-size to fill the container width — no fixed pixel size needed. The grid is fully fluid.

Level Calculation (recommended)

// Quantile-based bucketing from raw daily activity count
function getLevel(count, max) {
  if (count === 0) return 0;
  const ratio = count / max;
  if (ratio <= 0.15) return 1;
  if (ratio <= 0.35) return 2;
  if (ratio <= 0.55) return 3;
  if (ratio <= 0.80) return 4;
  return 5;
}

React Implementation

function getResponseTimeColor(hours: number | null): string {
  if (hours === null) return '#0F172A';  // default — no data
  if (hours < 1) return '#16A34A';    // green
  if (hours <= 4) return '#F59E0B';   // amber
  return '#EF4444';                    // red
}

function formatResponseTime(hours: number | null): string {
  if (hours === null) return '—';
  return hours.toFixed(1) + 'h';
}

function getResponseTimeSub(hours: number | null): string {
  if (hours === null) return 'no blocked/failed today';
  return 'to blocked/failed items';
}

10 Responsive Behavior

BreakpointBehavior
> 1024px Full layout: 52-column heatmap fills card width naturally. Cells approx 14–18px. All month labels visible.
768–1023px Main grid collapses to single column. Heatmap card becomes full-width — cells shrink proportionally (approx 10–14px). Still readable. Summary cards: 2×2 grid.
< 768px Cells shrink further (approx 6–10px). Consider: overflow-x: auto on .heatmap-grid with fixed minimum cell size if needed. Summary cards: single column. Month labels may overlap — consider showing every other month label.
< 480px Day labels column can be hidden. Heatmap becomes horizontally scrollable with min-width constraint.

11 Data Contract & Supabase

Heatmap Data Query

-- Daily activity count for team heatmap (last 52 weeks)
SELECT
  DATE(created_at) AS activity_date,
  COUNT(*) AS activity_count
FROM activity_logs
WHERE project_id = :project_id
  AND created_at >= NOW() - INTERVAL '52 weeks'
GROUP BY DATE(created_at)
ORDER BY activity_date ASC;

Expected Response Shape

// API returns sparse array (only days with activity)
interface HeatmapDay {
  activity_date: string;  // "2025-04-01"
  activity_count: number; // 0–N raw count
}

// Client fills in missing dates as 0 and maps to levels 0–5

Avg Response Time Query

-- Average response time to blocked/failed items today
SELECT
  AVG(
    EXTRACT(EPOCH FROM (first_response_at - status_changed_at)) / 3600
  ) AS avg_response_hours
FROM tc_executions
WHERE project_id = :project_id
  AND status IN ('blocked', 'failed')
  AND DATE(status_changed_at) = CURRENT_DATE
  AND first_response_at IS NOT NULL;
If query returns null (no blocked/failed items today), display the empty state: value = "—", sub = "no blocked/failed today".
