4 Add Result 모달 간소화 (조건부 UI)

Status 선택에 따라 필요한 필드만 자동으로 펼치고, 불필요한 필드는 접습니다. "Passed" 시에는 최소 필드만, "Failed" 시에는 상세 기록 필드를 자동 펼침. 비작동 리치 텍스트 도구 바를 제거하고 깔끔한 plain textarea로 교체합니다.

디자인 원칙

1. Escalation Path (단계적 상세화)

어떤 모드에서든 필요에 따라 상세 레벨을 올릴 수 있습니다:

Quick Status (1-click) → 인라인 팝업 (note) → Full Modal (all fields) → Focus Mode Pass/Retest Fail/Block "Open full modal" 대량 실행

2. Status-Driven UI (상태 기반 적응)

모달이 "Passed" 선택 시 간단해지고, "Failed" 선택 시 상세해지도록 UI가 반응합니다.

3. Keyboard-First (키보드 우선)

모든 Quick Status 동작에 키보드 단축키 제공. Focus Mode와 일관된 키 매핑.

4. Zero-Friction Pass (마찰 없는 Pass)

Pass는 1-클릭 또는 P 키 한 번이면 끝. 팝업도 모달도 없음.
