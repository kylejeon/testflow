1필터 옵션 정의

옵션 value 의미 데이터 범위 Delta 비교 기준
Active Run DEFAULT 'active_run' 현재 진행 중인 Run들 (status IN ('new','in_progress','paused','under_review'))에 포함된 test_results만 Active Run의 전체 기간 (Run 생성일~현재) 직전 완료된 Run(들)의 Pass Rate와 비교. 없으면 Delta 숨김
Last 7 Days '7d' 최근 7일 내 기록된 test_results created_at >= now() - 7d 이전 7일 (8~14일 전)과 비교 — 현재 로직과 동일
Last 30 Days '30d' 최근 30일 내 기록된 test_results created_at >= now() - 30d 이전 30일 (31~60일 전)과 비교
All Time 'all' 프로젝트 전체 기간의 test_results 날짜 필터 없음 Delta 숨김 — 비교 기준 없음

1.1 "Active Run" 상세 정의

Active Run이란? test_runs.status가 new, in_progress, paused, under_review 중 하나인 Run을 의미합니다. 이미 ProjectsContent.tsx Line 194에서 동일한 조건으로 Active Run count를 계산하고 있습니다.

왜 기본값인가? QA팀이 가장 관심 있는 것은 "지금 진행 중인 Run의 품질"입니다. 7일 기준은 임의적이지만, Active Run 기준은 실제 작업 컨텍스트와 일치합니다.

Active Run이 0개일 때: "No active runs" 메시지 표시 + Last 7 Days로 자동 fallback하지 않음 (사용자가 직접 다른 옵션 선택).

Delta 비교 — "직전 완료 Run": test_runs.status = 'completed'인 Run 중 updated_at DESC 기준 가장 최근 것(들)을 비교 대상으로 사용. Active Run이 여러 개면 모두 합산, 직전 완료 Run도 같은 수만큼 합산.

2.4 Dashboard 카드 동기화

Dashboard Pass Rate 카드는 Report 페이지와 동일한 기간 필터를 사용해야 합니다.

구현 방식: 필터 값을 localStorage 또는 URL search param에 저장. 기본값: 'active_run'.

카드 표시:

  • Active Run 선택 시: delta badge = "+2.1% vs prev run"
  • 7d 선택 시: delta badge = "+2.1% vs last week" (현재와 동일)
  • 30d 선택 시: delta badge = "+2.1% vs prev 30d"
  • All Time 선택 시: delta badge 숨김

카드 자체에 드롭다운 없음. Report 페이지에서 변경한 필터가 카드에도 반영됩니다.

3데이터 — 옵션별 Supabase 쿼리 로직

3.1 Active Run

// Step 1: Active Run IDs 가져오기
const { data: activeRuns } = await supabase
  .from('test_runs')
  .select('id, project_id, created_at')
  .in('project_id', projectIds)
  .in('status', ['new', 'in_progress', 'paused', 'under_review']);

const activeRunIds = (activeRuns ?? []).map(r => r.id);

// Step 2: Active Run의 test_results만 가져오기 (날짜 필터 없음)
const { data: results } = await supabase
  .from('test_results')
  .select('id, run_id, test_case_id, status, created_at')
  .in('run_id', activeRunIds)
  .order('created_at', { ascending: false });

// Step 3: Delta용 — 직전 완료 Run
const { data: prevRuns } = await supabase
  .from('test_runs')
  .select('id, project_id')
  .in('project_id', projectIds)
  .eq('status', 'completed')
  .order('updated_at', { ascending: false })
  .limit(activeRunIds.length || 1);  // Active Run 수만큼

3.2 Last 7 Days (현재 로직 유지)

const sevenDaysAgo = new Date(Date.now() - 7 * 24 * 60 * 60 * 1000).toISOString();
const fourteenDaysAgo = new Date(Date.now() - 14 * 24 * 60 * 60 * 1000).toISOString();

// Primary
.in('run_id', allRunIds).gte('created_at', sevenDaysAgo)
// Delta (prev 7d)
.gte('created_at', fourteenDaysAgo).lt('created_at', sevenDaysAgo)

3.3 Last 30 Days

const thirtyDaysAgo = new Date(Date.now() - 30 * 24 * 60 * 60 * 1000).toISOString();
const sixtyDaysAgo  = new Date(Date.now() - 60 * 24 * 60 * 60 * 1000).toISOString();

// Primary
.in('run_id', allRunIds).gte('created_at', thirtyDaysAgo)
// Delta (prev 30d)
.gte('created_at', sixtyDaysAgo).lt('created_at', thirtyDaysAgo)

7필터 값 저장 방식

방식: localStorage — key: 'testably_pr_period'

초기값: 'active_run' (localStorage에 값 없을 때)

동기화 흐름:

  • Report 페이지에서 드롭다운 변경 → localStorage.setItem() + setPeriod()
  • Dashboard 마운트 시 → localStorage.getItem()으로 현재 필터 읽기
  • 둘 다 같은 키를 참조하므로 항상 동기화됨

대안: URL search param (?period=7d) — Report 페이지 URL 공유 시 필터 유지 가능. 두 방식 중 Dev 판단에 맡김.

8Edge Cases

#케이스처리
1Active Run 0개Pass Rate = "—", 차트 빈 상태, 안내 메시지: "No active runs. Start a new run or select a different time period."
2선택 기간 내 test_results 0건Pass Rate = "—", Delta 숨김, 차트 all-zero 표시
3All Time + 데이터 매우 많음 (10,000+건)성능 주의: .limit(5000) + 클라이언트 사이드 집계. 향후 RPC 함수로 전환 가능.
4직전 완료 Run 없음 (Active Run Delta)Delta badge 숨김
5Active Run이 30일 이상 진행 중Daily Trend 차트를 주별로 자동 집계 (30일 초과 시)
6필터 변경 중 로딩현재 데이터 유지 + spinner overlay. 데이터 도착 후 교체.
