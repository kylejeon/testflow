📋 변경 요약: CEO 검토 결과, 기존 4개 서브탭(All/Active/Completed/Failed)에서 3개 서브탭 + Toolbar Result Filter로 변경 확정.

// 서브탭 정의 (v1.1)
const SUBTABS = [
  { key: 'all',       label: 'All',       icon: 'ri-list-check-3',          color: '#6366F1' },
  { key: 'active',    label: 'Active',    icon: 'ri-play-circle-fill',     color: '#3B82F6' },
  { key: 'completed', label: 'Completed', icon: 'ri-checkbox-circle-fill', color: '#22C55E' },
] as const;

// 필터 로직
const filterBySubtab = (runs: TestRun[], tab: string) => {
  if (tab === 'all') return runs;
  if (tab === 'active') return runs.filter(r =>
    ['new', 'in_progress', 'paused', 'under_review'].includes(r.status)
  );
  if (tab === 'completed') return runs.filter(r => r.status === 'completed');
  return runs;
};

// Badge count 계산

3 Task B: Result Filter 드롭다운 NEW

3-1. 필터 옵션 정의

#옵션 라벨필터 조건적용 탭비고
1All Results(필터 없음 — 기본값)All, Active, Completed드롭다운 기본 선택
2Has Failuresrun.failed > 0All, Completed (주 사용)기존 Failed 서브탭 대체
3All Passedrun.failed === 0 && run.blocked === 0All, Completed100% 통과 Run만
4Has Blockedrun.blocked > 0All, Active, Completed차단된 TC 포함 Run

3-2. 필터 적용 순서

파이프라인: 전체 runs → 서브탭 필터 → Result Filter → Search → Milestone Filter → Sort
Result Filter는 서브탭 필터 다음, 검색어 필터 이전에 적용됩니다.

3-3. 구현 코드 패턴

// Result Filter 타입 정의
type ResultFilter = 'all' | 'has_failures' | 'all_passed' | 'has_blocked';

const RESULT_FILTERS: { key: ResultFilter; label: string }[] = [
  { key: 'all',          label: 'All Results' },
  { key: 'has_failures', label: 'Has Failures' },
  { key: 'all_passed',   label: 'All Passed' },
  { key: 'has_blocked',  label: 'Has Blocked' },
];

// 필터 로직
const filterByResult = (runs: TestRun[], filter: ResultFilter) => {
  switch (filter) {
    case 'has_failures': return runs.filter(r => (r.failed || 0) > 0);
    case 'all_passed':   return runs.filter(r => (r.failed || 0) === 0 && (r.blocked || 0) === 0);
    case 'has_blocked':  return runs.filter(r => (r.blocked || 0) > 0);
    default:              return runs;
  }
};

// state 추가
const [resultFilter, setResultFilter] = useState<ResultFilter>('all');

// 전체 필터 파이프라인 (useMemo 권장)
const filteredRuns = useMemo(() => {
  let result = filterBySubtab(runs, activeTab);
  result = filterByResult(result, resultFilter);
  if (searchQuery) result = result.filter(r => r.name.toLowerCase().includes(searchQuery.toLowerCase()));
  if (milestoneFilter) result = result.filter(r => r.milestone_id === milestoneFilter);
  return sortRuns(result, sortKey);
}, [runs, activeTab, resultFilter, searchQuery, milestoneFilter, sortKey]);

⚠️ 주의: Result Filter 선택 시 badge count는 변하지 않습니다. Badge는 항상 서브탭의 전체 카운트를 표시합니다.
예: Completed 탭에서 "Has Failures" 선택 시 → badge는 여전히 15 (전체 completed), 실제 표시 리스트는 3개 (failed > 0인 Run만).
