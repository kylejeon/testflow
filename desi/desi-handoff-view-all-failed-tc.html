기존 테이블 컬럼 (현재)

☐ │ 테스트 케이스 │ 폴더 │ 우선순위 │ 상태 │ 담당자 │ 생성일 │ 작업

변경 후 테이블 컬럼 — status=failed일 때

☐ │ TC-ID │ Title │ Project │ Priority │ Fail Count │ Last Failed │ Last Run │ Assignee
컬럼 전환 규칙:
• status=all → 기존 컬럼 유지 (폴더, 상태, 생성일, 작업 포함)
• status=failed → 3개 컬럼 추가 (Fail Count, Last Failed, Last Run) + 불필요한 컬럼 숨김 (폴더, 생성일)
• status=passed/blocked/untested → 해당 상태에 맞는 적절한 컬럼 조합 (Phase 2에서 상세화)
