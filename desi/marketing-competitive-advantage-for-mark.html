1 최근 개선/기획된 혁신 기능 (14개)

아래는 최근 집중적으로 개발·기획된 기능들입니다. Testably 독자 표시는 경쟁사에 동등한 기능이 없는 항목입니다.

Focus Mode + Collapsible Sidebar Testably 독자

Run 리스트에서 1-click으로 Focus Mode 진입. 280px 접이식 사이드바로 TC 목록을 탐색하면서 테스트 실행. J/K 키보드 내비게이션, 자동 미실행 TC 시작 기능 포함.

경쟁사: TestRail·Zephyr·Qase 모두 별도 Focus Mode 없음. 테스트 실행 시 페이지 이동 필요.

Step-Level Pass/Fail 강화

테스트 스텝 단위로 개별 Pass/Fail/Blocked/Retest 상태 기록. Focus Mode 내에서 스텝별 인라인 상태 변경 + 코멘트 추가 가능.

경쟁사: TestRail은 스텝별 상태 지원하나 별도 화면 필요. Qase는 기본 지원, Zephyr는 제한적.

Discovery Log — 구조화된 탐색적 테스팅 Testably 독자

탐색적 테스팅 세션을 5단계 상태(Active → Paused → Review → Completed/Abandoned)로 관리. 구조화된 입력(Bug/Observation/Concern/Idea/Question), Severity·Category 태깅, 히트맵, 타이머 기반 세션 추적.

경쟁사: TestRail은 "Exploratory Session" 타입만 제공(구조화 없음). Zephyr·Qase에는 전용 탐색적 테스팅 모듈 없음.

Release Readiness Score Testably 독자

Project Overview에서 릴리즈 준비도를 0-100 점수로 자동 산출. Pass Rate, 미실행 TC 비율, Blocker 수, Milestone 진행률을 가중 합산. 색상 코딩(Green 80+/Amber 50-79/Red <50)으로 즉시 판단 가능.

경쟁사: 동등 기능 없음. TestRail은 커스텀 리포트로 유사 지표 구성 가능하나 자동 산출 불가.

My Runs + Priority 자동 산출 Testably 독자

Overview에서 나에게 할당된 Active Run을 Priority 순으로 자동 정렬. Priority는 Milestone 기한 기반 자동 산출(≤3일=Critical, 4-7일=High, 8-14일=Medium, >14일=Low) + 수동 Override 지원.

경쟁사: 기한 기반 자동 Priority 산출 기능 없음. 모두 수동 Priority 설정만 가능.

Milestone Detail 대폭 개선 강화

D-day 카운트다운, TC 요약(Total/Passed/Failed/Blocked), Pass Rate 게이지, Risk Indicator, Assignee 정보를 Detail Info Bar에 한눈에 표시. 4개 콘텐츠 탭(Results/Status/Activity/Issues) + 컬러 아이콘 서브탭.

경쟁사: TestRail 마일스톤은 기본 Progress bar만 제공. Zephyr는 "Test Cycle"로 관리(D-day 없음).

멤버 초대 복원 + Settings Members 탭 신규

Project Overview에서 1-click 멤버 초대 + Settings > Members 탭에서 프로젝트별 멤버 관리. 역할 기반 접근 제어(Admin/Member/Viewer), 구독 티어별 인원 제한 자동 체크.

경쟁사: 유사 기능 제공하나 프로젝트별 역할 관리 + 구독 티어 연동은 Testably가 더 세밀.

2-Row Unified Header Testably 독자

전체 6개 탭에 걸쳐 통일된 2-Row 헤더 시스템 적용. Row 1 = 56px 공유 ProjectHeader(6탭 내비게이션), Row 2 = 42px 서브탭(컬러 아이콘 + pill 뱃지 + 액션 버튼). 학습 비용 제로, 페이지 간 이동이 자연스러움.

경쟁사: TestRail은 탭별 레이아웃 불일치. Zephyr는 Jira 종속 UI. Qase만 비교적 일관된 UI.

서브탭 컬러 아이콘 시스템 신규

모든 서브탭에 상태별 컬러 코딩된 Remix 아이콘 적용. Active(파랑), Completed(초록), Overdue(빨강) 등 시각적으로 즉시 인지 가능. Pill 뱃지로 각 카테고리 카운트 표시.

경쟁사: 대부분 텍스트 전용 탭. Qase만 아이콘 일부 사용하나 컬러 코딩은 없음.

Documents 폴더 시스템 신규

Documents 탭에 계층형 폴더 시스템 도입. 232px 사이드바에 폴더 트리, 하위 폴더 지원, 드래그-앤-드롭 이동, 새 폴더 생성 모달. 프로젝트 문서를 체계적으로 조직.

경쟁사: TestRail은 Sections로 관리(폴더 UI 아님). Qase는 기본 폴더 지원. Zephyr는 Jira 연동 첨부.

TC Assignee 일괄 할당 강화

Test Cases 리스트에서 체크박스 다중 선택 → 일괄 Assignee 할당. 대규모 테스트 스위트를 팀원에게 빠르게 분배. Bulk Action 툴바로 상태 변경, 폴더 이동도 가능.

경쟁사: TestRail·Qase는 Bulk Edit 지원. Zephyr는 Jira 내 제한적 Bulk 작업.

Generate with AI 강화

테스트 케이스를 AI로 자동 생성. 요구사항이나 기능 설명을 입력하면 구조화된 TC(Steps + Expected Results)를 생성. 반복 작업을 대폭 줄여 QA 생산성 극대화.

경쟁사: Qase는 AIDEN으로 수동→자동 변환 중심. TestRail은 2025년 AI TC Generator 추가. Zephyr는 AI 기능 없음.

Runs & Results 리디자인 강화

3개 서브탭(All/Active/Completed) + Result Filter(All/Has Failures/All Passed/Has Blocked)로 Run 상태를 효율적으로 필터링. Priority 그룹핑으로 긴급한 Run을 최상단에 자동 배치.

경쟁사: TestRail은 테이블 뷰 중심(Priority 그룹핑 없음). Zephyr는 Jira 필터 의존.

키보드 단축키 시스템 신규

Focus Mode 내 완전한 키보드 워크플로우: J/K(TC 이동), P/F/B/R(상태 변경), C(코멘트), H(히스토리), [(사이드바 토글), /(검색). 마우스 없이 테스트 실행 완료 가능.

경쟁사: TestRail·Zephyr·Qase 모두 전용 키보드 단축키 시스템 없음.

3 핵심 차별점 딥다이브 — Top 6

마케팅 메시지로 가장 활용하기 좋은 6대 차별점을 깊이 정리합니다.

① Focus Mode — "테스트 실행의 재정의"

왜 중요한가: QA 엔지니어는 하루 업무 시간의 60% 이상을 테스트 실행에 사용합니다. 기존 도구들은 "TC 목록 페이지 → TC 상세 페이지 → 결과 입력 → 돌아가기"를 반복해야 합니다. 이 context-switching이 생산성을 떨어뜨립니다.

Testably 방식: Run 카드에서 ▶ 버튼 1-click → Focus Mode 진입. 좌측 280px 사이드바에서 TC 목록을 탐색하고, 우측에서 스텝별 실행 결과를 기록합니다. J/K로 다음/이전 TC 이동, P/F/B/R로 상태 변경. 마우스에서 손을 뗄 필요가 없습니다.

경쟁사 현황: TestRail, Zephyr Scale, Qase 모두 Focus Mode 개념 자체가 없습니다. 테스트 실행 시 반드시 페이지 이동이 발생합니다.

② Discovery Log — "탐색적 테스팅의 구조화"

왜 중요한가: 탐색적 테스팅은 스크립트 기반 테스팅이 놓치는 엣지 케이스를 발견하는 핵심 기법입니다. 하지만 대부분의 도구는 "자유 텍스트 노트"만 제공하여, 발견 사항이 체계 없이 흩어집니다.

Testably 방식: 전용 Discovery Log 탭에서 세션을 5단계 상태로 관리합니다. 발견 사항을 Bug/Observation/Concern/Idea/Question 5가지 타입으로 구조화하여 입력합니다. Severity와 Category 태깅, 세션 타이머, 커버리지 히트맵까지 제공하여 탐색적 테스팅의 효과를 측정 가능하게 만듭니다.

경쟁사 현황: TestRail은 "Exploratory Session" 테스트 케이스 타입만 제공하며 구조화된 입력이 없습니다. Zephyr Scale과 Qase에는 탐색적 테스팅 전용 모듈 자체가 없어, 별도 외부 도구(Rapid Reporter 등)를 사용해야 합니다.

③ Release Readiness Score — "릴리즈 결정의 데이터화"

왜 중요한가: "이번 릴리즈 내보내도 되나?"는 QA 리더가 매 스프린트 받는 질문입니다. 기존에는 여러 리포트를 수동으로 취합하여 주관적으로 판단했습니다.

Testably 방식: Project Overview에서 Release Readiness Score를 0-100 점수로 자동 산출합니다. Pass Rate(40%), 미실행 TC 비율(25%), Blocker 수(20%), Milestone 진행률(15%)을 가중 합산하며, 80+ Green(Go), 50-79 Amber(Caution), <50 Red(Block)으로 색상 코딩합니다.

경쟁사 현황: TestRail, Zephyr Scale, Qase 모두 동등한 자동 산출 기능이 없습니다. TestRail은 커스텀 리포트로 유사 지표를 구성할 수 있으나, 자동 산출·색상 코딩·점수화는 불가합니다.

④ Milestone 기한 기반 Priority 자동 산출 — "긴급한 일을 먼저"

왜 중요한가: QA 팀은 동시에 여러 Run을 진행합니다. 어떤 Run을 먼저 해야 하는지 판단하려면 Milestone 기한을 일일이 확인해야 합니다.

Testably 방식: Milestone 기한(end_date)으로부터 남은 일수를 자동 계산하여 Critical(≤3일)/High(4-7일)/Medium(8-14일)/Low(>14일) Priority를 자동 부여합니다. 필요 시 수동 Override도 가능(priority_override 컬럼). Runs 리스트에서 Priority 그룹핑으로 Critical → High → Medium → Low 순서로 자동 정렬합니다.

경쟁사 현황: TestRail, Zephyr Scale, Qase 모두 수동 Priority 설정만 가능합니다. 기한 기반 자동 Priority 산출 + 그룹핑 기능은 Testably 독자입니다.

⑤ AI 통합 (Generate with AI) — "AI가 TC를 작성합니다"

왜 중요한가: 테스트 케이스 작성은 반복적이고 시간 소모가 큽니다. 특히 새 기능이 추가될 때마다 수십 개의 TC를 수동으로 작성해야 합니다.

Testably 방식: 요구사항이나 기능 설명을 입력하면 AI가 구조화된 TC(Title, Steps, Expected Results)를 자동 생성합니다. 생성된 TC를 검토·수정하여 바로 테스트 스위트에 추가할 수 있습니다.

경쟁사 현황: Qase는 AIDEN으로 "수동 TC → 자동화 스크립트" 변환에 초점. TestRail은 2025년에 AI TC Generator를 추가했으나, 요구사항 기반 생성보다는 템플릿 기반. Zephyr Scale은 AI 기능이 없습니다.

⑥ 통일된 UX 패턴 — "배워야 할 것이 하나"

왜 중요한가: 테스트 관리 도구는 기능이 많을수록 학습 곡선이 급증합니다. 탭마다 UI가 다르면 팀 온보딩에 수주가 소요됩니다.

Testably 방식: 6개 전체 탭(Dashboard, Test Cases, Runs, Milestones, Documents, Discovery Log)에 동일한 2-Row Header + 42px 컬러 아이콘 서브탭 + Toolbar(Search + Filters + Sort) 패턴을 적용합니다. 하나의 탭 사용법을 배우면 나머지 5개 탭도 즉시 사용 가능합니다.

경쟁사 현황: TestRail은 탭별로 헤더와 서브탭 레이아웃이 다릅니다. Zephyr Scale은 Jira UI에 종속되어 자체 UX 일관성이 제한적입니다. Qase는 비교적 일관적이나 서브탭 패턴이 없습니다.
