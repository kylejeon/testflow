1. TC 생성 → Draft 상태 + Toast "Mark as Active" 링크

모든 TC는 생성 시 lifecycle_status: 'draft'로 시작합니다. 사용자가 즉시 Active로 전환할 수 있도록 Toast에 액션 링크를 포함합니다.

1.1 사용자 플로우

QuickCreateTCModal에서 TC 생성 완료
Toast 표시: "TC명" created as Draft. + Mark as Active 링크
사용자가 "Mark as Active" 클릭 → 즉시 Active로 전환
확인 Toast: "TC명" marked as Active. (초록 배경, 3초)

1.2 Toast 라이브 프리뷰

① 생성 직후 Toast (5초)
"User registration" created as Draft. Mark as Active
② "Mark as Active" 클릭 후 확인 Toast (3초)
"User registration" marked as Active.

3. Active/Draft → Deprecated 전환 — 확인 다이얼로그

Deprecated 전환은 보호 전환입니다 — 새 Run에 포함되지 않으므로 확인 다이얼로그로 실수를 방지합니다.

3.1 개별 TC Deprecated 전환 플로우

TC 상세 패널 → 상태 배지(Active/Draft) 클릭 → 드롭다운
'Deprecated' 선택 → 확인 다이얼로그 표시
사용자가 "Deprecate" 클릭 → 배지 색상 → 회색 전환. 리스트 행 opacity: 0.55
기본 필터(All)에서 사라짐 → Deprecated 탭에서만 보임
History 기록 + Toast: "테스트 케이스가 Deprecated로 변경되었습니다."

3.2 개별 TC 확인 다이얼로그 — 라이브 프리뷰

Deprecate Test Case?

"User registration with valid email"을 Deprecated로 변경합니다. 새 Run에 포함되지 않습니다.

현재 2개 Run에 포함되어 있으며, 기존 Run에는 영향 없습니다.

3.3 벌크 Deprecated 확인 다이얼로그 — 라이브 프리뷰

Deprecate 5 Test Cases?

5개 TC를 Deprecated로 변경합니다. 새 Run에 포함되지 않습니다.

이 중 3개가 진행 중인 Run에 포함되어 있습니다. 기존 Run에는 영향 없습니다.

4. Deprecated → Active/Draft 복원 플로우

Deprecated TC는 언제든 Active 또는 Draft로 복원할 수 있습니다. 복원은 안전한 전환이므로 확인 다이얼로그가 없습니다.

Deprecated 탭에서 복원할 TC 클릭 → 상세 패널
Deprecated 배지(회색) 클릭 → 드롭다운
"Active" 선택 → 즉시 전환 (확인 다이얼로그 없음)
배지 회색 → 초록 전환. opacity 0.55 → 1.0 복원
기본 필터(All)에서 다시 보임. 새 Run에 포함 가능.
History: "Kyle restored from Deprecated to Active"
Deprecated → Draft도 가능: 폐기된 TC를 수정하여 재활용할 때. 같은 플로우로 "Draft" 선택 가능.

7. 기존 Run — Deprecated TC 안내 배너

이미 생성된 Run에 포함된 TC가 이후에 Deprecated 되면, Run 상세 페이지에서 안내 배너를 표시합니다. 데이터는 절대 삭제하지 않습니다.

7.1 Run Detail 페이지 — 안내 배너

일부 TC가 Deprecated 상태입니다. 이 Run에 포함된 TC 2개가 이후에 Deprecated로 변경되었습니다. 실행 결과는 유지되며, 향후 새 Run에는 포함되지 않습니다.
