§1 — URL & Routing

Entry Point

URL/testcases?period=7d&sort=created_desc
TriggerOverview page "View All →" link in Recently Added section
Context detectionsearchParams.has('sort') && sort === 'created_desc'
Back link target/testcases-overview

URL Params (extends existing)

period'7d' (pre-selected) — existing param
sort ★ NEW'created_desc' | 'created_asc' | 'fail_count' | 'title'
status'all' default (not 'failed' like View All Failed TC)

Pattern Reuse (from View All Failed TC)

Failed TC URL/testcases?status=failed&period=7d
Recent TC URL/testcases?period=7d&sort=created_desc
Same pattern: Both views reuse the existing /testcases page with URL params to drive dynamic header, default filters, and column visibility. No new routes needed.

§6 — Status Tab Default Behavior

Entry Context → Default Active Tab

sort=created_desc"All" tab active (shows all statuses)
status=failed"Failed" tab active (existing behavior)
No params"All" tab active (existing behavior)
Rationale: "Recently Added" includes TCs of all statuses — the user wants to see everything new, not just failures. "All" is the correct default.
§7 — React Implementation

Header Config Extension

// TestCaseList.tsx — extend header config
const getHeaderConfig = (params: URLSearchParams) => {
  if (params.get('sort') === 'created_desc') {
    return {
      icon: 'ri-time-line',
      iconColor: '#6366F1',
      title: 'Recently Added Test Cases',
      subtitle: getSubtitleByPeriod(params.get('period')),
      badgeStyle: 'recent',  // bg #EEF2FF, color #4F46E5
      backLink: { text: 'Back to Overview', href: '/testcases-overview' },
      defaultStatus: 'all',
      showCreatedColumn: true,
      showFailColumns: false,
    };
  }
  if (params.get('status') === 'failed') {
    return { /* existing Failed TC config */ };
  }
  return { /* default config */ };
};

Sort URL Sync (new)

// Add sort state synced to URL
const [sortBy, setSortBy] = useSyncedParam('sort', 'fail_count');

// Column header click handler
const handleSortClick = (col: string) => {
  if (sortBy === col + '_desc') setSortBy(col + '_asc');
  else setSortBy(col + '_desc');
};
