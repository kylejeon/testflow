CI/CD Pipelines
Requires Professional or above
Connect your CI/CD pipeline to automatically report test results.
Register in your GitHub Secrets
TESTABLY_URL https://api.testably.app/functions/v1/upload-ci-results
TESTABLY_TOKEN testably_a1b2c3d4e5f6g7h8...
Registering both values as environment variables keeps them secure without modifying your code.
# .github/workflows/testably.yml name: Upload Test Results to Testably on: push: branches: [ main ] pull_request: branches: [ main ] jobs: test: runs-on: ubuntu-latest steps: - uses: actions/checkout@v3 - name: Run Tests run: npm test - name: Upload Results to Testably if: always() env: TESTABLY_URL: ${{ secrets.TESTABLY_URL }} TESTABLY_TOKEN: ${{ secrets.TESTABLY_TOKEN }} run: | curl -X POST "$TESTABLY_URL" \ -H "Authorization: Bearer $TESTABLY_TOKEN" \ -H "Content-Type: application/json" \ -d '{ "run_id": "YOUR_RUN_ID", "results": [ { "test_case_id": "SUI-1", "status": "passed", "note": "Test passed successfully", "elapsed": 1.5, "author": "GitHub Actions" } ] }'
How to use:
1. Register the environment variables in GitHub Secrets
2. Copy the YAML code and add to your CI/CD config
3. Set test_case_id to your test case ID (e.g. SUI-1)
4. Set status to passed/failed/blocked based on test outcome

1
Function-based approach — suitable for simple scripts
Collect results using the report_result() function and upload them all at once at the end.
# testably_report.py import requests, os TESTABLY_URL = os.environ.get("TESTABLY_URL") TESTABLY_TOKEN = os.environ.get("TESTABLY_TOKEN") results = [] def report_result(test_case_id, status, note="", elapsed=0): results.append({ "test_case_id": test_case_id, "status": status, # "passed"|"failed"|"blocked"|"retest" "note": note, "elapsed": elapsed, "author": "pytest" }) # Example def test_login(): report_result("SUI-1", "passed", note="Login test passed") def test_signup(): report_result("SUI-2", "passed", note="Signup flow verified") if __name__ == "__main__": test_login(); test_signup() resp = requests.post(TESTABLY_URL, headers={"Authorization": f"Bearer {TESTABLY_TOKEN}"}, json={"run_id": "YOUR_RUN_ID", "results": results}) print("Status:", resp.status_code)
2
conftest.py approach — suitable for pytest projects
Uses pytest hooks to automatically collect and upload results without modifying test code.
# conftest.py — save in your project root import pytest, requests, os, time TESTABLY_URL = os.environ.get("TESTABLY_URL") TESTABLY_TOKEN = os.environ.get("TESTABLY_TOKEN") RUN_ID = os.environ.get("TESTABLY_RUN_ID", "YOUR_RUN_ID") TEST_CASE_MAP = { "test_login": "SUI-1", "test_signup": "SUI-2", } _results, _timings = [], {} @pytest.hookimpl(hookwrapper=True) def pytest_runtest_call(item): start = time.time() yield _timings[item.name] = round(time.time() - start, 2) @pytest.hookimpl(tryfirst=True, hookwrapper=True) def pytest_runtest_makereport(item, call): outcome = yield report = outcome.get_result() if report.when == "call": tc_id = TEST_CASE_MAP.get(item.name) if not tc_id: return status = "passed" if report.passed else "failed" if report.failed else "blocked" _results.append({"test_case_id": tc_id, "status": status, "note": str(report.longrepr)[:300] if report.failed else "", "elapsed": _timings.get(item.name, 0), "author": "pytest"}) def pytest_sessionfinish(session, exitstatus): if _results and TESTABLY_TOKEN: requests.post(TESTABLY_URL, headers={"Authorization": f"Bearer {TESTABLY_TOKEN}"}, json={"run_id": RUN_ID, "results": _results})
How to use:
1. Register your token in the TESTABLY_TOKEN environment variable
2. Method 1: Pass the test case ID and result to report_result()
3. Method 2: Save conftest.py to your project root and map function names to IDs
4. Replace run_id with the Run ID from Testably
5. Status values: passed / failed / blocked / retest

API Token Scope Configuration Guide
When using a Personal Access Token (PAT) or Fine-grained Token in GitHub, only the following permissions are needed.

Required

Read organization info (for org repositories)
