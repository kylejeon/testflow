  assignee?.login 을 그대로 assignee_display_name 필드에 저장

  포함. 실명 (.name) 을 얻으려면 /users/{login} 별도 호출 필요

### 수정

#### _shared/github-user.ts (신규)
- GitHubUserNameCache = Map<login, name|null>
- getGitHubUserName(login, token, cache):
  - 캐시 hit → 즉시 반환 (fetch 생략)
  - miss → /users/{login} 호출, data.name 추출 (trim 후 빈 문자열이면 null)
  - 404/403/429 실패 → null 캐시 + 재시도 방지 (호출자가 login fallback)
- resolveAssigneeDisplayName(login, token, cache):
  - 실명 있으면 실명, 없으면 login, login 도 없으면 null

#### sync-github-metadata/index.ts
- import { createGitHubUserNameCache, resolveAssigneeDisplayName }
- for (const row of projRows) 진입 직전 nameCache 생성
  → 같은 project 내 모든 row/item 이 공유 (같은 assignee 중복 fetch 방지)

#### create-github-issue/index.ts
- 동일 import
- metadata 생성 시 resolveAssigneeDisplayName 로 이름 결정
- 이슈 생성은 1회 호출이라 cache scope = 요청 단위 (createGitHubUserNameCache() 즉시 생성)
