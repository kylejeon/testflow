### LinkedIn — Flaky Detection AI 소개 (4/15)

```
Flaky 테스트는 QA 팀의 가장 큰 고충 중 하나입니다.

같은 코드인데 랜덤하게 성공/실패를 반복하는 테스트.
무시하면 진짜 버그를 놓치고, 매번 확인하면 시간이 낭비됩니다.

Testably의 Flaky Detection AI는:

1. 통계적 패턴 분석으로 불안정 테스트 자동 탐지
2. 근본 원인 자동 분류 (타이밍, 환경, 데이터 의존성 등)
3. 심각도 스코어링으로 우선순위 결정
4. Jira 이슈 자동 생성 + TC ID 라벨 태깅

AI가 문제를 찾고 → 원인을 분석하고 → Jira까지 연결.
수동 작업 제로의 워크플로우입니다.

TestRail, Zephyr, qTest에는 없는 기능입니다.

데모 보기 → [링크]

#Testably #AI #FlakyTest #Jira #QA
```
