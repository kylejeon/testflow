Focus Mode — TC 리스트 사이드바

테스트 실행 시 전체 TC 리스트를 사이드바로 확인하고, 상태별 시각적 피드백을 제공하는 기능 기획

2026-03-27 Product — Kyle Feature: focus-mode-sidebar 다음 단계: Desi 디자인 핸드오프
Section 1

문제 정의 — 왜 사이드바가 필요한가

CEO Pain Point

"Focus Mode에서 전체 TC 리스트를 볼 수 없어서 답답하다. 현재 어디까지 진행했는지, 어떤 TC가 Failed인지 한눈에 파악이 안 된다."

현재 Focus Mode는 한 번에 하나의 TC만 표시하고, N 또는 J/K 키로 순차 이동만 가능합니다. 40개 이상의 TC가 있는 Run에서는 전체 진행 상황을 파악하기 어렵고, 특정 TC로 즉시 이동할 방법이 없습니다.

솔루션 요약

좌측에 접기/펼치기 가능한 TC 리스트 사이드바를 추가합니다. 각 TC는 실행 상태(Passed/Failed/Blocked/Retest/Untested)에 따른 컬러 아이콘을 표시하고, 현재 TC를 하이라이트하며, 클릭으로 즉시 이동할 수 있습니다. 상단에 진행률 바와 필터 칩을 배치합니다.

Section 3

사이드바 상세 스펙

3-1. 위치 및 크기

속성스펙
위치좌측(Left) — 좌→우 읽기 흐름에 맞춰 리스트를 먼저 스캔하고 우측 본문으로 시선 이동
기본 너비280px (최소 240px, 최대 360px 리사이즈 가능)
접기(Collapse) 시너비 0px — 완전히 숨기고 토글 핸들(12px 탭)만 좌측 가장자리에 표시
토글 방법① 토글 핸들 클릭 ② 키보드 [ (열기/닫기 토글) ③ 본문 영역을 넓게 쓰고 싶을 때 자유롭게 접기
기본 상태펼침(Open) — 사용자가 접으면 세션 내 기억 (localStorage)
반응형화면 너비 < 1024px 시 자동 접힘, 토글로 오버레이 열기
배경#FAFBFC — 메인 본문(#FFFFFF)과 미세한 구분
구분선우측에 1px solid #E2E8F0

3-2. 사이드바 내부 구조 (위→아래)

영역내용상세
① 진행률 헤더 세그먼트 프로그레스 바 + 카운트 "15 / 40 completed" 형태
세그먼트 바: 각 상태별 비율을 컬러로 표시 (Passed 초록, Failed 빨강, Blocked 노랑, Retest 보라, Untested 회색)
높이 6px, 둥근 모서리
② 필터 칩 상태별 퀵 필터 All · Passed · Failed · Blocked · Retest · Untested
활성 필터: Indigo(#6366F1) 배경 + 흰색 텍스트
클릭 시 해당 상태만 표시, 다시 클릭하면 해제
③ 검색 TC 검색 인풋 돋보기 아이콘 + placeholder "Search TC..."
TC-ID 또는 이름으로 실시간 필터링
④ TC 리스트 스크롤 가능한 TC 목록 각 항목: 상태 아이콘 + TC-ID + TC 이름
현재 TC 하이라이트
클릭으로 해당 TC로 이동
가상 스크롤 (100+ TC 대응)

Section 6

진행률 표시

사이드바 상단에 세그먼트형 프로그레스 바를 배치합니다. 기존 Focus Mode 헤더의 단일색 프로그레스 바를 대체하지 않고, 사이드바에 더 상세한 정보를 추가합니다.

요소스펙
텍스트 카운트 "15 / 40 completed" — 좌측 정렬
15: font-weight: 700; color: #6366F1 (Indigo, 강조)
/ 40 completed: font-weight: 500; color: #94A3B8
세그먼트 바 높이 6px, 둥근 모서리 3px
각 세그먼트: Passed(#22C55E) → Failed(#EF4444) → Blocked(#F59E0B) → Retest(#8B5CF6) → Untested(#E2E8F0)
비율에 따라 너비 자동 계산, 부드러운 전환 애니메이션
상태별 카운트 (옵션) 필터 칩 옆에 숫자 뱃지 표시 — 예: "Failed (3)"
0개인 상태는 뱃지 숨김

Section 10

기술 고려사항

항목상세
레이아웃 변경 기존 focus-overlay의 flex-direction: column 안에 flex-direction: row 영역을 추가하여 [Sidebar | Main] 구조 생성.
Header/Progress는 sidebar 위가 아닌 전체 상단에 유지.
상태 관리 기존 tests prop의 runStatus 필드를 사이드바가 참조.
onStatusChange 호출 후 부모 컴포넌트가 tests 배열을 업데이트하면 사이드바도 자동 반영.
가상 스크롤 기존 VirtualList.tsx 활용 가능. 100+ TC 시 성능 최적화.
아이템 높이 고정: 48px
localStorage 캐시 사이드바 열림/닫힘 상태: focusMode.sidebarOpen
사이드바 너비: focusMode.sidebarWidth
활성 필터: 세션 내에서만 유지 (새 Run 시 초기화)
애니메이션 사이드바 토글: width transition 200ms ease
상태 아이콘 변경: scale bounce 150ms
진행률 바: width transition 400ms ease
영향 범위 수정 파일: FocusMode.tsx (주요), FocusModeProps 인터페이스 변경 없음
새 컴포넌트: FocusSidebar.tsx 분리 권장
