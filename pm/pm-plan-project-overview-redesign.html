2-3. CEO 핵심 피드백

"각 항목마다 보여주는 정보가 너무 적다."

⚠️ 구체적 문제점 6가지

  • ① Summary 패널이 숫자 4개만 나열 — 40, 68%, 4, 2. 트렌드(↑↓), 전주 대비, 목표 대비 맥락 전무
  • ② My Tasks가 단순 카드 리스트 — 우선순위 정렬 없음, 기한(Due) 없음, 모듈 분류 없음. 40개를 스크롤해야 함
  • ③ Pass Rate 68%가 좋은 건지 나쁜 건지 판단 불가 — 목표 대비 갭, 전 Run 대비 변화, 추이 그래프 없음
  • ④ 릴리스 준비 상태 지표 없음 — "지금 릴리스 가능한가?" 판단 불가
  • ⑤ 테스트 실행 진행 상황 없음 — 4개 Run의 개별 진행률, 예상 완료 시각 없음
  • ⑥ 모듈별/기능별 상태 브레이크다운 없음 — 어느 모듈에 문제가 집중되는지 알 수 없음

    Q2"내가 지금 뭘 해야 하나?"내 할당 TC, 우선순위 정렬, 기한 임박 항목부분우선순위 정렬, Due Date, 필터/그루핑

    Q4"어디에 문제가 집중되나?"모듈별/폴더별 Pass/Fail 분포미제공모듈별 분석 전무

    4-3. 경쟁사에서 배울 점

    Best Practices 3가지

    ① TestRail — "Run 중심 진행률": 프로젝트 Overview에서 각 Run의 Pass/Fail/Untested를 수평 스택바로 한눈에 보여줌. Testably의 "Runs: 4"보다 훨씬 정보량이 많음.

    ② Qase — "맥락 있는 숫자": 모든 통계 카드에 작은 스파크라인 + 전기간 대비 변화율(↑12%)을 함께 표시. 숫자 하나만 보여주지 않음.

    ③ PractiTest — "커스텀 Alert": 사용자가 "Pass Rate < 70%이면 경고", "미실행 TC > 20개면 경고" 같은 조건을 직접 설정. Attention Needed의 기준을 투명하게 만듦.

    5-2. 위젯별 상세 개선 사항

    Release Readiness 신규

    위치: 좌측 메인 최상단 (가장 눈에 띄는 자리)

    구성: 종합 점수(0~100) + 신호등(🟢🟡🔴) + 4개 구성 지표 (Pass Rate vs 목표, Critical Bugs, 미실행 TC, Milestone %)

    신호등 기준: 🟢 Score ≥ 80 → Go / 🟡 60~79 → Conditional / 🔴 < 60 → Not Ready

    Q1 "릴리스 가능한가?" 직접 답변

    Summary 패널 고밀도화

    현재: TC 40 / Pass Rate 68% / Runs 4 / Bugs 2 (숫자만)

    개선: 각 숫자 옆에 ↑↓ 변화량 + vs 기준(전주/전 Run) + 미니 스파크라인(7일 추이). Pass Rate 도넛에 목표선 표시.

    Q5 "전보다 나아지고 있나?" 답변 보조

    My Runs CEO 확정 변경

    현재: My Tasks — 나에게 assign된 TC 40개 플랫 리스트

    변경: My Tasks → My Runs — TC 단위가 아닌 Run 단위로 표시 (CEO 피드백 반영)

    • Priority 자동 산출: Milestone 기한 기반 — ≤3일 🔴Critical / 4~7일 🟠High / 8~14일 🟡Medium / >14일 🟢Low. Milestone 미연결 Run은 Low.

    • 수동 Override: 필요 시 사용자가 직접 priority 변경 가능 (DB: priority_override 컬럼)

    • Run 카드: Run명 + 3색 진행률 바(Pass/Fail/Remaining) + 퍼센트 + Milestone Due + 담당자 + ▶ Continue 버튼

    • 헤더: "My Runs (3)" — 나에게 할당된 Active Run 수

    Q2 "내가 뭘 해야 하나?" 직접 답변 + Focus Mode 바로 진입

    Run Progress 신규

    위치: 우측 사이드바 (Summary 아래)

    구성: 활성 Run 리스트, 각각: Run명 + 미니 수평 바(Pass/Fail/Remaining) + 퍼센트 + 완료 여부 체크

    8-1. Priority 산출 로직 (확정)

    Priority 결정 공식
    priority = priority_override ?? computeFromMilestone(milestone.end_date)

function computeFromMilestone(end_date):
  if (!end_date) return 'low'           // Milestone 미연결 → Low
  days_remaining = diffDays(end_date, today)
  if (days_remaining <= 3)  return 'critical'
  if (days_remaining <= 7)  return 'high'
  if (days_remaining <= 14) return 'medium'
  return 'low'
