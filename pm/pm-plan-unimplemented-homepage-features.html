3. P0 Jira 양방향 동기화 (Bidirectional Sync)

F-001: Jira ↔ Testably 양방향 상태 동기화

Jira에서 이슈 상태가 변경되면 Testably의 관련 테스트 결과에 반영되고, Testably에서 상태 변경 시 Jira 이슈도 업데이트. 현재는 Testably→Jira 수동 이슈 생성만 가능.

홈페이지 주장
"sync status bidirectionally", "Bidirectional status sync"
현재 상태
Testably→Jira 단방향 이슈 생성만 (run-detail에서 수동 버튼)
예상 공수
5일 (프론트 2일 + 백엔드 3일)
디자인 필요
Yes — 동기화 상태 표시 UI, Settings 매핑 화면

3-1. 구현 범위

A. Jira → Testably (인바운드)

항목상세
Jira Webhook 수신Supabase Edge Function jira-webhook-handler 신규 생성. Jira Cloud webhook이 POST로 이슈 업데이트 이벤트 전송 → 함수에서 수신/파싱
이벤트 매핑Jira 이슈 transition (e.g. To Do → In Progress → Done) → Testably test_results.issues 배열에서 해당 이슈 key 검색 → 연관 TC의 상태를 업데이트할지 사용자에게 알림
DB 스키마jira_sync_log 테이블 신규: id, project_id, jira_issue_key, jira_status, testably_run_id, testably_tc_id, synced_at, direction ('inbound'|'outbound')
보안Jira webhook secret 검증 (X-Hub-Signature 헤더)

B. Testably → Jira (아웃바운드 상태 동기화)

항목상세
상태 변경 트리거test_results에 새 결과 INSERT 시, 해당 TC에 linked Jira issue가 있으면 Jira REST API로 transition 호출
상태 매핑 설정Settings → Jira Integration에 상태 매핑 UI 추가: Testably "Passed" → Jira "Done", "Failed" → "Reopened" 등 사용자 설정 가능
Edge Functionsync-jira-status 신규: Jira REST API POST /rest/api/3/issue/{key}/transitions

C. Settings UI 변경

// Settings → Jira Integration 탭에 추가할 섹션 // 1. Webhook URL 표시 (복사 가능) — Jira Admin에 등록하라는 가이드 // 2. Status Mapping 테이블 (Testably Status ↔ Jira Transition) // 3. Sync Direction 토글: Inbound only / Outbound only / Bidirectional // 4. Sync Log 테이블 (최근 동기화 이력) interface JiraSyncMapping { testably_status: 'passed' | 'failed' | 'blocked' | 'retest'; jira_transition_id: string; jira_transition_name: string; } // DB: jira_settings 테이블에 컬럼 추가 ALTER TABLE jira_settings ADD COLUMN webhook_secret TEXT, ADD COLUMN sync_direction TEXT DEFAULT 'outbound', ADD COLUMN status_mappings JSONB DEFAULT '[]';
