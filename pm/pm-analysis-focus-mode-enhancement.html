§4. PM 분석 — CEO 의견에 대한 견해

✅ PM 견해: CEO 의견에 전적으로 동의하며, 추가 제안 3건을 더합니다

CEO가 지적한 모든 항목(Expected Result, 첨부 사진, TC ID, Folder, Priority, Tags, Comments)은 경쟁사 분석에서도 테스트 실행 모드의 표준 기능으로 확인됩니다. 현재 Focus Mode는 "미니멀한 실행 뷰"로 설계되었으나, 실제 테스터가 다른 화면으로 돌아가지 않고도 테스트를 완료할 수 있는 "자급자족(Self-contained) 실행 환경"이 되어야 합니다.

4-1. CEO 요청 항목별 동의 분석

CEO 요청PM 의견근거

§5. 정보 우선순위 및 레이아웃 제안

5-1. 정보 3-Tier 구분

Focus Mode는 "집중 실행 환경"입니다. 정보 과부하를 방지하면서도 자급자족이 되려면 정보 우선순위에 따른 시각적 계층 구분이 필요합니다.

🎯 Tier 1 — 메인 콘텐츠 (항상 표시)
  • Test Steps + Expected Result (각 step별)
  • Precondition (존재 시)
  • Attachments / Screenshots (인라인 썸네일)
  • Status 버튼 (P/F/B/R/S)
  • Note 입력
📋 Tier 2 — 메타데이터 바 (헤더 아래 컴팩트)
  • TC-ID (TC-001 포맷)
  • Priority (색상 뱃지)
  • Folder (텍스트)
  • Tags (뱃지 목록)
  • Assignee (아바타)
💬 Tier 3 — 확장 패널 (접이식, 필요 시 펼침)
  • Comments (이전 댓글 목록)
  • Execution History (이전 Run 결과 이력)
  • Description (긴 설명 — 접힘 가능)

§8. 핵심 디자인 원칙

원칙설명적용
Focus First Focus Mode의 핵심 가치는 "집중"입니다. 정보를 추가하되 복잡도를 높이면 안 됩니다. Tier 1(메인)은 항상 표시, Tier 3(보조)는 접이식으로 필요 시만 펼침
Keyboard First 기존 키보드 단축키 UX는 Testably의 차별화 포인트입니다. 새 기능 추가 시 키보드 접근성 유지. Tier 3 패널 토글에 단축키 추가 (C=Comments, H=History)
Self-Contained 테스터가 Focus Mode를 나가지 않고 필요한 모든 정보를 확인할 수 있어야 합니다. 첨부파일 라이트박스, 댓글 인라인 표시, 실행 이력 패널
Progressive Disclosure 핵심 정보부터 보여주고, 상세 정보는 사용자의 명시적 액션으로 드러냅니다. Metadata Bar (항상 표시, 한 줄) → 접이식 패널 (클릭 시 펼침)
