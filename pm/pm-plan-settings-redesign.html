1-2. 발견된 문제점

문제 ①: "General" 탭이 사실상 "Billing" 탭
General 탭에 구독 플랜/가격 비교/Upgrade CTA만 있음. "일반 설정"에 기대하는 기능(언어, 테마, 시간대 등)이 전혀 없음. 탭 이름과 실제 내용의 불일치.
문제 ②: Jira 연동이 General에도 있었음 (Desi 목업)
Desi의 settings-redesign.html에서는 General 탭에 "Jira Connection" 섹션이 있고, Integrations 탭에도 중복 표시. 실제 코드에서는 Integrations에만 존재. → 혼란 유발.
문제 ③: CI/CD와 Integrations 경계 모호
CI/CD 탭의 "API Tokens"는 사실상 CI/CD만을 위한 것이 아니라 범용 API 토큰. 웹훅(Integrations)과 API 토큰(CI/CD)이 분리되어 있어, "외부 연동"이라는 하나의 개념이 2개 탭에 분산.
문제 ④: Danger Zone이 Desi 목업에만 존재
Desi 목업 Profile 탭에 "Delete Account" 버튼이 있으나, 실제 코드에는 미구현. → 기획 확정 후 구현 필요.
문제 ⑤: 누락 기능 다수
언어 설정, 테마(라이트/다크), 시간대, 날짜 형식, 데이터 내보내기/가져오기, API Key 관리(범용), Account Deletion 등이 Settings에 없음.

3-3. 제안 근거 상세

① "General" → "Billing" 이름 변경: 현재 General 탭에는 구독 플랜 카드, 6티어 비교표, Monthly/Annual 토글, Upgrade CTA만 있습니다. 이것은 "Billing & Subscription"이지 "General"이 아닙니다. TestRail과 Qase도 Billing은 별도 이름으로 분리합니다.

② Preferences 탭 신설: 사용자가 "General Settings" 또는 "Preferences"에서 기대하는 기능들(언어, 테마, 시간대)이 현재 어디에도 없습니다. Testably는 이미 한국어/영어 i18n을 지원(src/i18n/)하므로, 전환 UI만 추가하면 됩니다.

③ Integrations + CI/CD 통합: 현재 Integrations(Jira + Webhook)와 CI/CD(토큰 + 가이드)가 분리되어 있지만, 사용자 관점에서 "외부 서비스 연동"은 하나의 개념입니다. 하나의 탭 안에서 섹션 카드로 구분하면 탐색이 쉬워집니다. 단, API 토큰은 범용성이 있으므로 별도 탭으로 분리.

④ API & Tokens 분리: CI/CD에 묻혀 있던 API 토큰을 독립 탭으로 승격합니다. API 토큰은 CI/CD뿐 아니라 커스텀 스크립트, 외부 도구 연동, 자동화에 사용됩니다. Qase의 "Apps" 개념과 유사.
