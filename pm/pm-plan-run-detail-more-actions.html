2
CEO 결정 필요: 삭제 vs 기능 구현
Decision Required

Option A: 버튼 삭제 (권장)

이유: 현재 run-detail 페이지에는 이미 충분한 액션 버튼들이 존재합니다 (Focus Mode, Quick Status, Add Result, Create Jira Issue, Detail Panel). "..." 버튼이 없어도 핵심 워크플로우에 지장이 없으며, dead code를 유지하는 것은 UX 품질을 떨어뜨립니다.

작업량: 5분 (버튼 JSX 2곳 삭제)

Option B: 기능 구현 (PM 권장)

이유: Run Detail에서 자주 사용하지만 현재 접근이 불편한 기능들을 모아놓으면 사용성이 크게 향상됩니다. 경쟁 QA 도구(TestRail, Zephyr, qTest)에서도 Run Detail에 More Actions 메뉴를 제공합니다.

작업량: 1~2시간

PM 의견: 버튼이 이미 UI에 존재하고, 사용자가 클릭하고 있을 것이므로 Option B (기능 구현)을 권장합니다. 아래에 구현할 메뉴 항목을 기획합니다.

UI 구현 스펙

드롭다운 메뉴 디자인

  • 위치: 버튼 아래 오른쪽 정렬 (absolute positioning)
  • 배경: white, border: 1px solid #E2E8F0, shadow-lg
  • 너비: w-56 (14rem)
  • 각 항목: px-3 py-2 text-[0.8125rem], hover 시 bg-[#F8FAFC]
  • 구분선: border-t border-[#E2E8F0] my-1
  • Delete 항목: text-[#EF4444]
  • 바깥 클릭 시 닫힘: useRef + useEffect 패턴
  • Escape 키로 닫힘

필요한 State 추가

const [showRunMenu, setShowRunMenu] = useState(false); const runMenuRef = useRef<HTMLDivElement>(null); // 바깥 클릭 감지 useEffect(() => { const handler = (e: MouseEvent) => { if (runMenuRef.current && !runMenuRef.current.contains(e.target as Node)) { setShowRunMenu(false); } }; document.addEventListener('mousedown', handler); return () => document.removeEventListener('mousedown', handler); }, []);
