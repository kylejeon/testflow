1-2. 불일치 원인 분석

근본 원인: 두 가지 다른 관점이 혼재

현재 시스템에는 두 가지 서로 다른 추상화 레벨이 섞여 있습니다:

관점설명해당 상태사용 맥락
A. 결과 상태
Outcome Status		 세션이 "끝났을 때" 어떤 결론인지. 리스트에서 전체 세션을 빠르게 분류할 때 필요. Open / Resolved / Dismissed 리스트 테이블, 필터, 통계
B. 운영 상태
Operational Status		 세션을 "실행하는 동안" 실시간으로 무슨 상태인지. 타이머 연동, 작업 흐름 제어에 필요. Not Started / Active / Paused / Passed / Failed / Blocked / Dismissed 세션 상세 페이지, 타이머

핵심 문제: 리스트 화면의 "Open"이 Detail의 어떤 상태에 매핑되는지 정의되지 않았습니다. "Open"은 Not Started인지, Active인지, Paused인지 알 수 없고, "Resolved"는 Passed만 의미하는지 Failed도 포함하는지 불명확합니다. 이 매핑 부재가 CEO에게 혼란을 준 직접적 원인입니다.

1-3. 구체적 충돌 지점 3가지

#충돌 지점리스트 관점디테일 관점문제
1"Open"의 범위Open = 아직 끝나지 않은 세션Not Started, Active, Paused 3개가 모두 "끝나지 않은" 상태Open이 3개 중 어디에 대응하는지 불명확
2"Resolved"의 범위Resolved = 완료된 세션Passed, Failed, Blocked 3개가 모두 "완료"될 수 있음Failed도 Resolved인가? Blocked는?
3용어 불일치"Open/Resolved" — 이슈 트래커 용어"Not Started/Active/Passed/Failed" — 테스트 실행 용어같은 제품에서 두 가지 용어 체계 혼용
