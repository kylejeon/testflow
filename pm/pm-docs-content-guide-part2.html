페이지 도입부 텍스트
Testably의 계정 설정에서 프로필 정보를 수정하고, 구독 요금제를 관리하며, 결제 내역을 확인할 수 있습니다. 이 가이드에서는 계정 생성부터 요금제 선택, 결제 흐름, 계정 삭제까지 전체 과정을 안내합니다.

Section 6.1 — 계정 생성 및 로그인

Testably 계정을 만드는 방법과 로그인 옵션을 설명합니다.

본문 콘텐츠
계정 생성 방법
Testably에 접속하면 Sign Up 페이지에서 새 계정을 만들 수 있습니다. 두 가지 방법을 지원합니다:

1. 이메일 회원가입
이메일 주소와 비밀번호를 입력해 계정을 생성합니다. 가입 즉시 사용할 수 있으며, 이메일 인증을 통해 계정을 활성화합니다.

2. Google 소셜 로그인
"Continue with Google" 버튼을 클릭하면 Google 계정으로 바로 로그인/가입할 수 있습니다. 별도의 비밀번호 설정이 필요 없습니다.

로그인
기존 계정이 있다면 Log In 페이지에서 이메일/비밀번호 또는 Google 로그인으로 접속합니다.
Tip
Google 소셜 로그인으로 가입한 경우 별도의 비밀번호가 설정되어 있지 않으므로, 비밀번호 변경 섹션 대신 Google 계정 보안 설정을 안내합니다.

Section 6.2 — 프로필 관리

이름, 아바타, 비밀번호를 수정하는 방법을 설명합니다.

6.2.1 이름 변경

본문 콘텐츠
Settings > Profile 탭에서 표시 이름(Display Name)을 변경할 수 있습니다.

변경 방법:
1. 좌측 사이드바 또는 상단 프로필 메뉴에서 Settings를 클릭합니다.
2. Profile 섹션의 Display Name 입력 필드에서 이름을 수정합니다.
3. "Save Changes" 버튼을 클릭하면 즉시 반영됩니다.

변경된 이름은 프로젝트 멤버 목록, 테스트 결과 작성자, 코멘트 등 Testably 전체에 표시됩니다.

6.2.2 아바타 변경

본문 콘텐츠
Testably는 두 가지 아바타 방식을 지원합니다:

1. 이모지 아바타 (기본)
계정 생성 시 기본 이모지 아바타(🐶)가 설정됩니다. Profile 섹션에서 이모지 선택 팔레트를 열어 원하는 이모지로 변경할 수 있습니다.

2. 사진 업로드
프로필 아바타 영역을 클릭하면 이미지 파일을 업로드할 수 있습니다. 지원 형식은 JPEG, PNG, WebP이며, 최대 파일 크기는 2MB입니다. 업로드된 이미지가 이모지 아바타보다 우선 표시됩니다.

6.2.3 비밀번호 변경

본문 콘텐츠
이메일로 가입한 사용자는 Settings > Profile 하단의 Change Password 섹션에서 비밀번호를 변경할 수 있습니다.

변경 방법:
1. New Password 필드에 새 비밀번호를 입력합니다.
2. Confirm Password 필드에 동일한 비밀번호를 다시 입력합니다.
3. "Update Password" 버튼을 클릭하면 비밀번호가 변경됩니다.

비밀번호 입력 필드 우측의 눈 아이콘을 클릭하면 입력 내용을 표시/숨김 전환할 수 있습니다.
Note
Google 소셜 로그인으로 가입한 사용자에게는 비밀번호 변경 섹션이 표시되지 않습니다. 비밀번호 관리는 Google 계정 설정에서 진행해주세요.

Section 6.3 — 구독 요금제

Testably의 6가지 요금제와 각 요금제별 기능 비교를 설명합니다.

본문 콘텐츠 — 도입부
Testably는 팀 규모와 필요에 맞는 다양한 요금제를 제공합니다. 무료 플랜으로 시작하여 팀이 성장하면 언제든 업그레이드할 수 있습니다. Settings > Billing 탭에서 현재 요금제를 확인하고, "View All Plans" 버튼으로 전체 요금제를 비교할 수 있습니다.

Annual Discount
연간(Annual) 결제를 선택하면 월별 가격 대비 15% 할인이 적용됩니다. 예를 들어 Professional 요금제는 월 $99 대신 약 $84/월(연 $1,008)로 이용할 수 있습니다. 요금제 선택 시 Monthly / Annual 토글로 결제 주기를 변경할 수 있습니다.
Viewer 무료 정책
Viewer 역할의 팀원은 좌석(Seat) 수에 포함되지 않습니다. 즉, Viewer는 무료로 무제한 초대할 수 있습니다. 팀 멤버 제한은 Admin과 Member 역할에만 적용됩니다. 읽기 전용 접근이 필요한 이해관계자에게는 Viewer 역할을 부여하여 비용 부담 없이 프로젝트 현황을 공유하세요.

Section 6.4 — 결제 및 Invoice History

Paddle 결제 흐름과 청구서(Invoice) 관리를 설명합니다.

본문 콘텐츠
결제 시스템
Testably는 Paddle을 결제 플랫폼으로 사용합니다. 요금제 업그레이드 시 Paddle 체크아웃 화면이 표시되며, 신용카드 또는 PayPal로 결제할 수 있습니다. Paddle이 세금 계산과 영수증 발행을 자동으로 처리합니다.

결제 흐름:
1. Settings > Billing에서 "Upgrade" 버튼 클릭 또는 요금제 비교 모달에서 원하는 요금제 선택
2. Paddle 체크아웃 오버레이가 화면에 표시됨
3. 결제 수단(신용카드/PayPal) 선택 후 결제 완료
4. 결제 성공 시 즉시 요금제가 업그레이드되며, 프로필에 새 요금제 뱃지가 표시됨

결제 주기:
Monthly(월간) 또는 Annual(연간) 결제를 선택할 수 있습니다. Annual 결제 시 15% 할인이 적용됩니다. Paddle Price ID는 각 요금제/결제주기 조합별로 별도 관리됩니다.

현재 Paddle 지원 요금제: Starter, Professional, Enterprise S, Enterprise M (Monthly/Annual 각각). Enterprise L은 "Contact Sales" 방식입니다.
본문 콘텐츠 — Invoice History
청구서 내역
Settings > Billing 하단의 Invoice History 섹션에서 과거 결제 내역을 확인할 수 있습니다.

표시 정보: 결제일, 설명(요금제명 — Monthly/Annual), 결제 금액, 상태(Paid), 다운로드 버튼

무료 플랜 사용자: "No invoices yet. Upgrade to a paid plan to see your billing history." 안내 메시지가 표시됩니다.

유료 플랜 사용자: 최근 결제 내역이 테이블 형태로 표시됩니다. 각 행의 다운로드 아이콘을 클릭하면 PDF 영수증을 받을 수 있습니다.
Trial (무료 체험)
일부 요금제는 무료 체험 기간을 제공할 수 있습니다. 체험 기간 중에는 Billing 탭의 Current Plan 카드에 "Trial ends in X days" 메시지가 표시됩니다. 체험 기간이 만료되면 자동으로 Free 플랜으로 전환됩니다.

Section 6.5 — 계정 삭제 (Danger Zone)

계정 영구 삭제 절차와 주의사항을 설명합니다.

본문 콘텐츠
Settings > Profile 하단에 Danger Zone 영역이 있습니다. 이곳에서 Testably 계정을 영구적으로 삭제할 수 있습니다.

⚠ 경고: 계정을 삭제하면 되돌릴 수 없습니다. 모든 데이터가 영구적으로 제거됩니다.

삭제 절차

1
"Delete Account" 버튼 클릭
Danger Zone 영역의 빨간 "Delete Account" 버튼을 클릭합니다.
2
이메일 주소 확인 입력
확인 모달이 표시됩니다. 계정에 등록된 이메일 주소를 정확히 입력해야 삭제가 활성화됩니다. 이 단계는 실수로 삭제하는 것을 방지합니다.
3
최종 삭제 확인
이메일이 일치하면 "I understand, delete my account" 버튼이 활성화됩니다. 클릭하면 계정이 즉시 삭제되고 로그인 페이지로 이동합니다.
삭제 시 제거되는 데이터
계정 삭제 시 다음 데이터가 영구적으로 제거됩니다: 프로필 정보, 생성한 프로젝트(소유자인 경우), 테스트 케이스, 테스트 런 기록, Discovery Log 세션, 마일스톤 데이터, 업로드된 첨부파일. 삭제 전에 Settings > Data Export에서 데이터를 백업하는 것을 권장합니다.

Page 7 — Keyboard Shortcuts

전역 단축키, 네비게이션, 포커스 모드, 페이지별 단축키
/docs/keyboard-shortcuts
페이지 도입부 텍스트
Testably는 마우스 없이도 빠르게 작업할 수 있도록 다양한 키보드 단축키를 제공합니다. 어느 페이지에서든 ? 키를 누르면 현재 페이지에서 사용 가능한 단축키 목록을 확인할 수 있습니다. 이 가이드에서는 모든 단축키를 카테고리별로 정리합니다.
Mac / Windows 표기
이 문서에서 Cmd는 macOS의 Command(⌘) 키를, Ctrl은 Windows/Linux의 Control 키를 의미합니다. Testably는 운영체제를 자동 감지하여 단축키 도움말에 적절한 기호를 표시합니다. 또한 한글/중국어 입력 중(IME 조합 상태)에는 단축키가 비활성화되어 오동작을 방지합니다.

Section 7.1 — 전역 단축키 (Global)

모든 페이지에서 사용할 수 있는 단축키입니다.

구현 소스
src/hooks/useKeyboardShortcuts.ts — 전역 키보드 이벤트 리스너
src/components/KeyboardShortcutsHelp.tsx — 단축키 도움말 모달 (컨텍스트별 필터링)
단축키 동작 설명
? 단축키 도움말 열기 현재 페이지에서 사용 가능한 단축키 목록을 오버레이로 표시합니다.
Cmd+K Command Palette 열기 어느 페이지에서든 빠른 검색 및 명령 실행 창을 엽니다. 입력 필드에 포커스가 있어도 동작합니다.
Cmd+Shift+F Focus Mode 진입 Test Run 상세 페이지에서 Focus Mode를 시작합니다.

Section 7.2 — 네비게이션 단축키 (Go to...)

페이지 간 빠른 이동을 위한 Chord(연속 입력) 패턴 단축키입니다. G 키를 누른 후 1초 이내에 두 번째 키를 누르면 해당 페이지로 이동합니다.

단축키 이동 대상 설명
G → T Test Cases 현재 프로젝트의 테스트 케이스 목록 페이지로 이동합니다.
G → R Runs & Results 현재 프로젝트의 테스트 런 목록 페이지로 이동합니다.
G → D Discovery Logs 현재 프로젝트의 디스커버리 로그 목록으로 이동합니다.
G → M Milestones 현재 프로젝트의 마일스톤 목록으로 이동합니다.
G → P Projects 프로젝트 목록(홈) 페이지로 이동합니다.
Chord 패턴 사용법
G를 누르면 1초간 대기 상태에 들어갑니다. 이 시간 안에 두 번째 키를 누르면 이동이 실행됩니다. 1초가 지나면 대기 상태가 해제됩니다. 텍스트 입력 필드에 포커스가 있을 때는 동작하지 않습니다.

Section 7.3 — 리스트 네비게이션 & 선택

테스트 케이스 목록, 테스트 런 목록 등 리스트/테이블 형태의 모든 페이지에서 사용합니다.

단축키 동작 설명
↑ / ↓ 항목 이동 리스트에서 이전/다음 항목으로 포커스를 이동합니다.
J / K 항목 이동 (Vim 스타일) J = 다음, K = 이전. 화살표 키와 동일하게 동작합니다.
Enter 항목 열기 / 확장 포커스된 항목의 상세 페이지를 열거나 아코디언을 확장합니다.
Space 체크박스 토글 포커스된 항목의 선택 체크박스를 켜거나 끕니다.
Cmd+A 전체 선택 현재 표시된 모든 항목을 선택합니다. 다시 누르면 전체 해제됩니다.
Shift+↑/↓ 범위 선택 Shift를 누른 채 위/아래로 이동하며 여러 항목을 연속 선택합니다.
Esc 선택 해제 / 닫기 선택된 항목을 해제하거나 열린 모달/오버레이를 닫습니다.

Section 7.4 — Focus Mode 단축키

Focus Mode는 테스트 실행 중 빠른 결과 기록을 위한 전용 모드입니다. 단일 키 입력으로 즉시 동작합니다.

구현 소스
src/components/FocusMode.tsx — useEffect 내 keydown 이벤트 리스너, 단일 키 바인딩
단축키 동작 설명
P Passed + 다음 이동 현재 테스트를 Passed로 마킹하고 자동으로 다음 테스트로 이동합니다.
F Failed + 다음 이동 현재 테스트를 Failed로 마킹하고 다음으로 이동합니다.
B Blocked + 다음 이동 현재 테스트를 Blocked로 마킹하고 다음으로 이동합니다.
R Retest + 다음 이동 현재 테스트를 Retest로 마킹하고 다음으로 이동합니다.
S Skip (Untested) + 다음 이동 현재 테스트를 건너뛰고(Untested 유지) 다음으로 이동합니다.
N 노트 추가 현재 테스트에 메모를 입력할 수 있는 노트 필드를 활성화합니다.
← / K 이전 테스트 이전 테스트 케이스로 돌아갑니다.
→ / J 다음 테스트 다음 테스트 케이스로 이동합니다.
Esc Focus Mode 종료 Focus Mode를 닫고 테스트 런 상세 페이지로 돌아갑니다.
Power User Tip
Focus Mode에서 P P P F P와 같이 연속으로 키를 눌러 빠르게 여러 테스트의 결과를 기록할 수 있습니다. 각 키 입력마다 자동으로 다음 테스트로 넘어가므로 대량의 테스트를 빠르게 처리할 수 있습니다.

Section 8.1 — 일반 (General)

Q1. Testably는 어떤 서비스인가요?
Testably는 소프트웨어 QA 팀을 위한 테스트 관리 플랫폼입니다. 테스트 케이스 작성, 테스트 런 실행, 마일스톤 추적, 탐색적 테스트(Discovery Log), Jira 연동, CI/CD 통합 등 QA 워크플로우 전체를 하나의 도구에서 관리할 수 있습니다. AI 기반 테스트 케이스 자동 생성 기능도 제공합니다.
Q2. 무료로 사용할 수 있나요?
네, Testably Free 플랜은 영구 무료입니다. 최대 3개 프로젝트, 3명의 팀원, 월 5회 AI 생성을 지원합니다. 소규모 팀이나 개인 프로젝트에 적합합니다. 더 많은 기능이 필요하면 Starter($49/월) 이상의 유료 플랜으로 업그레이드할 수 있습니다.
Q3. 팀원은 몇 명까지 초대할 수 있나요?
요금제별로 다릅니다: Free(3명), Starter(5명), Professional(20명), Enterprise S(50명), Enterprise M(100명), Enterprise L(100명+). 이 제한은 Admin과 Member 역할에만 적용됩니다. Viewer 역할은 좌석 수에 포함되지 않으므로 무제한으로 초대할 수 있습니다.
Q4. 연간 결제 시 할인이 있나요?
네, Annual(연간) 결제를 선택하면 월별 가격 대비 15% 할인이 적용됩니다. 예를 들어 Professional 요금제는 월 $99 → 약 $84/월로 이용할 수 있습니다. Settings > Billing에서 결제 주기를 변경할 수 있습니다.
Q5. 다른 테스트 관리 도구에서 데이터를 가져올 수 있나요?
네, Testably는 TestRail, Zephyr, Qase 등 주요 테스트 관리 도구에서 데이터를 가져올 수 있습니다. 테스트 케이스 목록 페이지에서 "Import" 버튼을 클릭하면 CSV 또는 XML 형식으로 기존 데이터를 import할 수 있습니다. 또한 대시보드의 온보딩 배너에서 "Import from TestRail" 기능으로 빠르게 마이그레이션할 수 있습니다.

Section 8.2 — 계정 & 로그인

Q6. 비밀번호를 잊어버렸어요. 어떻게 재설정하나요?
로그인 페이지에서 "Forgot your password?" 링크를 클릭하세요. 가입 시 사용한 이메일 주소를 입력하면 비밀번호 재설정 링크가 이메일로 전송됩니다. 링크를 클릭하여 새 비밀번호를 설정하면 됩니다. Google 소셜 로그인으로 가입한 경우에는 Google 계정의 비밀번호 관리를 이용해주세요.
Q7. Google 로그인으로 가입했는데, 이메일/비밀번호로 전환할 수 있나요?
현재 Google 소셜 로그인과 이메일/비밀번호 로그인 간 전환은 지원되지 않습니다. Google 로그인으로 가입한 계정은 계속 Google 인증을 사용하게 됩니다. 이메일/비밀번호 로그인이 필요한 경우 새 계정을 생성해주세요.
Q8. 계정을 삭제하면 데이터는 어떻게 되나요?
계정 삭제는 되돌릴 수 없으며 모든 데이터가 영구적으로 제거됩니다. 삭제되는 데이터: 프로필 정보, 생성한 프로젝트(소유자인 경우), 테스트 케이스, 테스트 런 기록, Discovery Log 세션, 마일스톤 등. 삭제 전에 Settings > Data Export에서 JSON 또는 CSV로 데이터를 백업하는 것을 강력히 권장합니다.

Section 8.3 — 기능 관련

Q9. AI 테스트 케이스 생성은 어떻게 사용하나요?
테스트 케이스 목록 페이지에서 "AI Generate" 버튼을 클릭합니다. 두 가지 모드를 선택할 수 있습니다: (1) Text-based — 기능 설명을 텍스트로 입력하면 AI가 테스트 케이스를 자동 생성합니다. (2) Session-based — Discovery Log 세션을 선택하면 해당 탐색적 테스트 기록을 기반으로 테스트 케이스를 생성합니다 (Professional 이상). 생성 횟수는 요금제별로 제한됩니다: Free(5회/월), Starter(30회/월), Professional(150회/월), Enterprise(무제한).
Q10. Focus Mode란 무엇인가요?
Focus Mode는 테스트 런 실행 시 집중적으로 결과를 기록하기 위한 전용 UI입니다. 테스트 런 상세 페이지에서 "Focus Mode" 버튼을 클릭하거나 Cmd+Shift+F를 누르면 진입합니다. 왼쪽에 테스트 케이스 목록, 오른쪽에 상세 정보와 결과 입력 영역이 표시됩니다. 키보드 단축키(P=Passed, F=Failed, B=Blocked, R=Retest, S=Skip)로 빠르게 결과를 기록할 수 있습니다.
Q11. Jira 연동은 어떻게 설정하나요?
Settings > Integrations 탭에서 Jira Integration 섹션을 찾습니다. Jira 도메인(예: yourteam.atlassian.net), 이메일, API 토큰을 입력하고 연결합니다. 연결 후 테스트 결과에서 직접 Jira 이슈를 생성하거나 기존 이슈를 링크할 수 있습니다. Jira 연동은 Starter 요금제 이상에서 사용할 수 있습니다.
Q12. CI/CD 파이프라인과 연동할 수 있나요?
네, Professional 요금제 이상에서 CI/CD 통합을 지원합니다. Settings > CI/CD Pipelines 탭에서 API 토큰을 생성하고, GitHub Actions, GitLab CI, 또는 Python 스크립트로 테스트 결과를 자동으로 보고할 수 있습니다. 각 CI 플랫폼별 YAML 코드 스니펫이 제공됩니다.

Section 8.4 — 문제 해결 (Troubleshooting)

일반적인 문제와 해결 방법입니다.

T1. 로그인이 안 됩니다.
이메일/비밀번호 로그인: 비밀번호를 다시 확인해주세요. Caps Lock이 켜져있지 않은지 확인합니다. 비밀번호를 잊었다면 "Forgot your password?" 링크로 재설정하세요.

Google 로그인: Google 계정에 먼저 로그인되어 있는지 확인합니다. 브라우저의 팝업 차단기가 Google 인증 창을 막고 있을 수 있으므로 Testably 도메인의 팝업을 허용해주세요.

공통: 브라우저 캐시와 쿠키를 삭제한 후 다시 시도합니다. 다른 브라우저에서 시도해봅니다. 문제가 지속되면 고객 지원에 문의하세요.
T2. 결제했는데 요금제가 업그레이드되지 않았습니다.
Paddle 결제 완료 후 요금제 반영은 보통 즉시 이루어지지만, 드물게 네트워크 지연으로 몇 분이 걸릴 수 있습니다.

해결 방법:
1. 페이지를 새로고침(F5)합니다.
2. 로그아웃 후 다시 로그인합니다.
3. 5분 이상 지나도 반영되지 않으면 결제 확인 이메일(Paddle 발송)을 확인하고, hello@testably.app으로 해당 이메일을 전달하여 지원을 요청합니다.
T3. 테스트 케이스를 실수로 삭제했습니다. 복구할 수 있나요?
현재 Testably는 삭제된 테스트 케이스의 자동 복구(Undo) 기능을 제공하지 않습니다. 테스트 케이스 삭제 시 확인 모달이 표시되므로, 삭제 전에 한 번 더 확인해주세요. 중요한 데이터는 정기적으로 Settings > Data Export에서 JSON 또는 CSV로 백업하는 것을 권장합니다.
T4. Jira 연동이 작동하지 않습니다.
확인사항:
1. Jira 도메인이 올바른지 확인합니다 (예: yourteam.atlassian.net, https:// 제외).
2. 이메일이 Jira 계정과 동일한지 확인합니다.
3. API 토큰이 유효한지 확인합니다. Atlassian 계정 설정에서 새 토큰을 생성하고 다시 입력해보세요.
4. Jira 프로젝트에 접근 권한이 있는지 확인합니다.
5. Starter 요금제 이상인지 확인합니다 (Free 플랜에서는 Jira Link만 지원).

위 확인을 모두 마쳤는데도 문제가 지속되면 고객 지원에 문의하세요.
T5. 페이지가 느리거나 로딩되지 않습니다.
해결 방법:
1. 인터넷 연결을 확인합니다.
2. 브라우저를 최신 버전으로 업데이트합니다 (Chrome, Firefox, Safari, Edge 권장).
3. 브라우저 캐시를 삭제합니다 (Cmd+Shift+Delete 또는 Ctrl+Shift+Delete).
4. 브라우저 확장 프로그램을 일시 비활성화합니다 (특히 광고 차단기).
5. 다른 브라우저에서 시도합니다.
6. 문제가 지속되면 브라우저 개발자 도구(F12) > Console 탭의 에러 메시지를 캡처하여 고객 지원에 전달합니다.

Section 8.5 — 지원 요청 방법

FAQ에서 답을 찾지 못한 경우 Testably 팀에 직접 문의하는 방법입니다.

본문 콘텐츠
위 FAQ에서 문제 해결이 되지 않았거나 추가 도움이 필요한 경우, 아래 방법으로 Testably 팀에 문의할 수 있습니다.

1. 이메일 지원
hello@testably.app으로 이메일을 보내주세요. 문제 상황을 상세히 설명하고, 가능하면 스크린샷이나 에러 메시지를 첨부해주시면 더 빠른 해결이 가능합니다.

2. 앱 내 지원
Testably 하단의 "Contact" 링크 또는 Settings 페이지의 Support 섹션에서 지원 요청 폼을 이용할 수 있습니다.

응답 시간:
• Free / Starter: 커뮤니티 지원 — 업무일 기준 1-3일 이내 응답
• Professional: 우선 지원 — 업무일 기준 24시간 이내 응답
• Enterprise: 전담 지원 — SLA에 따른 보장된 응답 시간
효과적인 지원 요청 작성법
지원 요청 시 다음 정보를 포함하면 빠른 해결에 도움이 됩니다:

Keyboard Shortcuts 페이지 구현 노트:

• Mac에서는 ⌘ ⇧ ⌥ 기호 사용, Windows에서는 Ctrl, Shift, Alt 텍스트 표시
• navigator.platform 또는 navigator.userAgentData로 OS 감지 (기존 KeyboardShortcutsHelp.tsx 로직 재사용)
• <kbd> 태그에 별도 CSS 스타일링 (border, shadow, rounded로 키캡 느낌)
• 각 단축키 카테고리별 섹션 분리, 현재 컨텍스트와 무관하게 모든 단축키를 한 페이지에 나열

FAQ 페이지 구현 노트:

• 아코디언(Collapsible) 컴포넌트로 Q&A 구현 — 질문 클릭 시 답변 펼침/접힘
• 카테고리별 그룹핑: 일반, 계정 & 로그인, 기능 관련, 문제 해결
• 검색 기능 추가 고려: 상단에 텍스트 검색 필드 → FAQ 제목/본문 필터링
