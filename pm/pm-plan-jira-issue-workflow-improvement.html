2 UX Confusion Points

The sidebar Issues tab buttons "Link Issue" / "Link an issue" (DetailPanel.tsx lines 897, 948) open a Create Jira Issue modal, not a search/link interface. This is the exact confusion CEO identified.

3-4. "Link Existing Issue" Feature Detail

1
"Link Existing" click → inline input field expands within the tab (not a modal)
2
Enter Jira issue key (e.g., PROJ-456) → press Enter or click "Link"
3
(Optional) Verify existence: Jira API call to confirm issue exists. Show title preview if found, warning if not.
4
On confirm: Add key to latest test_result's issues[]. If no result exists, show message: "Please add a test result first" instead of silently creating a failed result.
Side-effect removal (fixes issue D): Remove the current logic that auto-creates a status: 'failed' result when creating/linking an issue from the Issues tab with no existing result. Replace with a user-friendly message: "This test case has no results yet. Please add a result first."
