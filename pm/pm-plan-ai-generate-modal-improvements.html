Critical Gap
발견 1: Jira Issue 탭은 현재 존재하지 않음
AIGenerateModal.tsx를 전수 분석한 결과, 모달에는 2가지 모드만 존재합니다:
• Text Input (mode: 'text') — 텍스트 입력으로 TC 생성
• Session-Based (mode: 'session') — Discovery Log 세션 선택으로 TC 생성

"Jira Issue" 탭/모드는 코드에 전혀 구현되어 있지 않습니다. 홈페이지 마케팅에서는 "Jira issue → test cases"가 pill 태그로 표시되어 있어 고객에게 약속된 기능이지만, 실제로는 미구현 상태입니다.

근거:
• useState<'text' | 'session'>('text') — 'jira' 모드 없음 (line 45)
• grep -n "jira\|Jira" AIGenerateModal.tsx — 결과 0건
• 경쟁사 비교 페이지 zephyr.ts에서만 "generate from... Jira issues" 마케팅 문구 존재
발견 2: Discovery Log 세션 선택은 이미 구현됨
CEO가 요청한 "드롭다운이나 리스트에서 세션을 선택하는 방식"은 이미 구현되어 있습니다:
• fetchSessions() — Supabase에서 프로젝트의 전체 세션 목록을 조회 (line 100-113)
• 세션 리스트를 라디오 버튼 UI로 표시 — 이름, mission, 상태(completed/active) 포함
• 선택 후 session_id를 Edge Function에 전달하여 AI가 세션 내용을 분석

단, Professional 이상에서만 사용 가능 (Free/Starter에서는 잠금 표시됨). CEO가 Free/Starter 계정으로 테스트했다면 이 UI를 보지 못했을 수 있습니다.

개선안 1 — Jira Issue 탭 신규 개발

신규 개발 필요

1.1 CEO 기대사항

Jira 이슈 번호(예: PROJ-123)를 입력하면, Testably가 자동으로 Jira API에 접근하여 티켓의 제목, 설명, Acceptance Criteria 등을 읽어와 AI 테스트 케이스를 생성하는 방식.

1.2 제안하는 동작 방식

Option A: 이슈 번호 입력 → 자동 Fetch (CEO 선호)

1
Jira Issue 모드 선택
모달의 3번째 모드 카드 "Jira Issue" 클릭. Jira 연동이 되어있지 않으면 "Settings에서 Jira를 먼저 연결하세요" 안내 표시.
2
이슈 번호 입력
텍스트 입력 필드에 Jira 이슈 키를 입력 (예: PROJ-123, QA-456). 복수 이슈 지원: 쉼표 또는 줄바꿈으로 여러 이슈 입력 가능 (예: PROJ-123, PROJ-124, PROJ-125).
3
"Fetch" 버튼 클릭 → Jira API 호출
백엔드(Edge Function)에서 사용자의 저장된 Jira credentials로 Jira REST API를 호출:
GET /rest/api/3/issue/{issueKey}?fields=summary,description,customfield_*

가져오는 필드: summary(제목), description(설명), acceptance criteria(커스텀 필드), priority, labels, components, subtasks
4
Fetch 결과 미리보기
가져온 Jira 이슈 정보를 카드 형태로 미리보기 표시:
• 이슈 키 + 제목 (링크 포함)
• 설명 텍스트 (첫 3줄 미리보기)
• Acceptance Criteria (있는 경우)
• Priority, Labels

사용자가 내용을 확인하고 "Generate Test Cases" 클릭.
5
AI 생성 → 기존 Step 3/4/5 플로우
Jira 이슈 내용이 Edge Function의 input_text 대신 jira_content로 전달됨. AI가 이슈 설명 + Acceptance Criteria를 기반으로 TC 제목을 생성. 이후 플로우는 기존과 동일 (제목 선택 → 상세 생성 → 저장).

Option B: Jira 이슈 검색/선택 (고급 버전, 향후 확장)

향후 고려 — Phase 2
이슈 번호를 직접 입력하는 대신, Jira 프로젝트 내 이슈를 검색하거나 필터링하여 선택하는 방식:
• 연결된 Jira 프로젝트의 이슈 목록을 드롭다운으로 표시
• JQL 검색 지원 (예: project = QA AND status = "To Do")
• 여러 이슈를 체크박스로 선택하여 한번에 TC 생성

이 방식은 Phase 1 완료 후 사용자 피드백을 반영하여 구현합니다.

1.3 전제조건 및 에러 처리

상황처리UI
Jira 연동 안 됨 모드 카드에 🔒 표시 + "Connect Jira first" 안내 카드 클릭 시 Settings → Integrations로 이동 링크 표시
이슈 번호 잘못됨 Jira API 404 → "Issue not found" 에러 입력 필드 하단 빨간 에러 메시지 + 이슈 키 형식 안내
Jira 토큰 만료/권한 없음 Jira API 401/403 "Jira credentials expired. Please reconnect in Settings." 안내
이슈 설명 없음 (제목만 있는 이슈) 제목만으로 AI 생성 시도, 경고 표시 "This issue has no description. Test cases will be based on the title only." 노란 경고
복수 이슈 중 일부 실패 성공한 이슈만 표시, 실패 이슈는 에러 목록 "3 of 5 issues fetched successfully. Failed: PROJ-999 (not found), PROJ-888 (no access)"

개선안 2 — Discovery Log 세션 선택 UX 개선

이미 구현됨 + UX 개선

2.1 현재 상태 (이미 구현된 부분)

CEO 요청사항은 이미 구현되어 있음
CEO가 원하는 "기존 세션을 리스트에서 선택하는 방식"은 이미 정확히 구현되어 있습니다:
• fetchSessions() — Supabase에서 프로젝트 세션 목록 조회
• 라디오 버튼 리스트 — 세션 이름, mission, 상태(completed/active) 표시
• 선택 후 session_id를 Edge Function에 전달

CEO가 이 UI를 보지 못한 가능성:
• Free/Starter 계정으로 테스트 → Session-Based 모드가 🔒 잠금 상태
• 모드 카드에 "Requires Professional or above" 표시만 보이고 내부 UI를 확인할 수 없음
• 또는 홈페이지 마케팅의 3가지 pill(Plain Text / Jira Issue / Discovery) 기준으로 기대했을 수 있음

2.2 현재 UI 구조

Session-Based 모드 Input 화면 (현재)
• "Select Session" 레이블
• 스크롤 가능한 리스트 (max-h-64 overflow-y-auto)
• 각 세션: 라디오 버튼 + 세션 이름 (bold) + mission (gray) + 상태 뱃지 (completed: 초록, 기타: 회색)
• 세션 없으면: "No sessions found for this project." 빈 상태
• 로딩 중: 스피너 + "Loading sessions..." 텍스트

2.3 UX 개선 제안

현재 구현은 기본적으로 동작하지만, 다음과 같은 개선으로 UX를 강화할 수 있습니다:

개선 A: 세션 미리보기 정보 확장

현재

● 세션 이름

● mission (1줄)

● 상태 뱃지

개선 후

● 세션 이름

● mission (1줄)

● 상태 뱃지

● 로그 항목 수 (예: "12 entries")

● 유형별 카운트 (예: "3 bugs, 5 notes, 4 steps")

● 생성일

● 소요 시간 (예: "45min")

개선 B: 필터/정렬 옵션

필터 UI 추가
• 상태 필터: All / Completed / In Progress — completed 세션이 AI 생성에 더 적합하므로 기본값을 "Completed"로 설정
• 정렬: 최신순 (기본, 이미 적용됨) / 로그 수 많은 순 — 로그가 많을수록 AI 입력이 풍부
• 검색: 세션 이름/mission으로 텍스트 검색 — 세션이 많은 프로젝트에서 유용

개선 C: 선택 후 요약 표시

세션 선택 후 하단에 요약 카드 표시
세션을 선택하면 하단에 미리보기 요약 카드가 나타남:

선택된 세션: "Login Flow Exploration"
로그 항목: 15개 (Bug 4, Note 6, Step 3, Blocked 2)
소요 시간: 1시간 23분
예상 생성 TC 수: ~8-12개

이를 통해 사용자가 AI 생성 전에 세션의 풍부함을 판단할 수 있음.

개선 D: 복수 세션 선택 (향후)

Phase 2 — 향후 확장
현재 라디오 버튼(단일 선택) → 체크박스(복수 선택)로 변경하여 여러 세션의 내용을 합쳐서 TC를 생성하는 옵션. 예: "Login Exploration" + "Payment Flow Exploration" 2개 세션 → 통합된 TC 세트 생성.
