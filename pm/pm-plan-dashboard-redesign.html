1 Dashboard의 핵심 목적 재정의

1-1. Dashboard가 답해야 할 핵심 질문

PM이나 QA 리드가 Dashboard를 열었을 때, 30초 안에 다음 질문들에 대한 답을 얻을 수 있어야 합니다:

6가지 핵심 질문
#질문필요 데이터현재 제공 여부
Q1"지금 릴리스할 수 있나?"전체 Pass Rate, 미실행 TC 수, Critical 버그 수, Blocker 이슈 현황부분 제공 Pass Rate만 있음
Q2"테스트 진행 상황이 어떤가?"활성 Run 진행률, 일일/주간 실행 트렌드, 완료 예상 시각부분 제공 Active Runs 수만 있음
Q3"어디에 문제가 집중되어 있나?"모듈별 실패율, 최근 실패한 TC 목록, 반복 실패 TC미제공
Q4"팀이 잘 움직이고 있나?"팀원별 활동량, 일일 TC 실행 수, 병목 지점미제공
Q5"탐색적 테스팅에서 뭐가 나왔나?"활성 세션 수, 발견 버그 수, 세션 커버리지부분 제공 Open Discoveries만 있음
Q6"지난주 대비 나아졌나?"주간 Pass Rate 변화, 버그 발견/해결 비율, 테스트 속도 추이미제공

⚠️ CEO 피드백 핵심: "각 항목마다 보여주는 정보가 너무 적다"

  • 통계 카드가 단일 숫자만 보여줌 → 트렌드, 비교, 맥락 없음
  • 차트가 1개뿐 → 다양한 관점의 분석 불가
  • Activity Feed가 단순 리스트 → 필터, 유형 구분, 시간대 분석 없음
  • 모듈별/팀원별 브레이크다운 전무 → "어디에 문제가 있나" 답 불가
  • 릴리스 준비 상태 지표 없음 → "릴리스 가능한가" 판단 불가
  • 비교 분석 (전주 대비, 전 Run 대비) 없음 → 개선/악화 판단 불가

2 Overview 탭 분석 + 개선안

2-1. 현재 → 개선 통계 카드

현재 4개 카드를 6개 고밀도 카드로 확장하고, 각 카드에 트렌드/맥락 정보를 추가합니다.

Pass Rate 개선

현재: 87.5% + "↑ 3.2% vs last run"

개선: 87.5% + 스파크라인(7일 추이) + "vs 목표 95%: 7.5% gap" + 전주 대비 ↑/↓ + Passed/Failed/Blocked/Untested 미니 스택바

Active Runs 개선

현재: 3 + "2 in progress · 1 queued"

개선: 3개 Run 각각의 이름 + 진행률 미니바 + 예상 완료 시각 + 담당자

Test Cases 개선

현재: 40 + "↑ 12% vs last week"

개선: 40 (Active 39 + Draft 1) + 상태별 도넛 차트 + 이번 주 신규 TC 수 + 자동화율 (Automated vs Manual 비율)

Discovery 개선

현재: 6 + "↑ 2 since yesterday"

개선: Open 6 / Resolved 7 / Total 15 + 이번 주 발견 버그 수 + 가장 활발한 세션 이름

Release Readiness 신규

신규 추가: 릴리스 준비도 점수 (0~100%) + 신호등 (🟢🟡🔴) + Critical/Blocker 버그 수 + 미실행 TC 수 + 마일스톤 진행률

Bug Snapshot 신규

신규 추가: 전체 버그 수 (Open/Resolved/Closed) + 심각도 분포 미니 차트 + 이번 주 신규 vs 해결 비율 + 평균 해결 시간

2-2. 차트 영역 확장

현재 1개 차트 + 1개 Activity 리스트를 4개 차트 위젯으로 확장합니다.

Test Execution Trend 개선

현재: 7일 바 차트 (실행 수만 표시)

개선: 스택 에어리어 차트 — Passed(녹) + Failed(적) + Blocked(황) + Untested(회) 일별 누적. 기간 셀렉터 (7d/14d/30d/Custom). 호버 시 상세 툴팁.

Module Health Map 신규

폴더(모듈)별 Pass/Fail 비율 트리맵 또는 히트맵. 크기 = TC 수, 색상 = Pass Rate. 문제 모듈을 시각적으로 즉시 식별.

Run Progress Overview 신규

활성 Run들의 실시간 진행률 비교. 수평 바 차트로 각 Run의 Passed/Failed/Remaining 비율 표시. 예상 완료 시각 포함.

Bug Trend (Open vs Resolved) 신규

일별 신규 버그 발견 수 vs 해결 수 라인 차트. 갭이 벌어지면 ⚠️ 경고. 버그 누적 추이도 함께 표시.

2-3. 추가 위젯 — 상세 정보 패널

Attention Needed 신규

즉시 조치 필요 항목 리스트: 반복 실패 TC (3회+), Blocked TC, 기한 초과 마일스톤, 7일 이상 미실행 TC. 각 항목 클릭 시 해당 페이지로 이동.

Milestone Tracker 신규

활성 마일스톤 리스트: 이름 + 진행률 바 + 남은 일수 + TC 상태 요약. 기한 임박 시 경고 색상.

5-2. Testably 차별화 전략

3가지 차별화 포인트:

① 릴리스 준비도 스코어: TestRail의 예측 기능을 넘어, Pass Rate + Critical 버그 + 미실행 TC + 마일스톤 진행률을 종합한 단일 점수(0~100)와 신호등 표시. 경쟁사 중 이를 단일 지표로 제공하는 곳 없음.

② Module Health Map: 폴더(모듈)별 테스트 상태를 트리맵/히트맵으로 시각화. 문제 모듈을 1초 만에 식별. Qase가 유사한 시도를 했으나 Testably가 더 직관적으로 구현 가능.

③ Discovery 통합 대시보드: 구조적 테스팅(TC/Run)과 탐색적 테스팅(Discovery)을 하나의 Dashboard에서 동시에 모니터링. 경쟁사들은 이 두 영역을 분리해서 보여줌.

6-5. 핵심 원칙

Dashboard 개선 3대 원칙

① 30초 룰: Dashboard를 열고 30초 안에 "프로젝트 건강 상태"에 대한 답을 얻을 수 있어야 한다. 모든 카드/차트는 이 기준으로 정보량을 설계한다.

② 맥락 있는 숫자: 단일 숫자만 보여주지 않는다. 반드시 트렌드(↑↓), 비교 기준(vs 목표/vs 전주), 상태 분포(미니 차트)를 함께 제공한다.

③ 점진적 공개: Overview에서 요약 → Analytics에서 심화 → 개별 탭(TC/Runs/Discovery)에서 상세. 같은 데이터를 다른 깊이로 보여준다.
