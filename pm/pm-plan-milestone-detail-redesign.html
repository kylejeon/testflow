2 공통 UI 패턴 적용 — 갭 분석

Testably의 공통 디자인 패턴(2-Row Unified Header, 42px 서브탭 Row, Toolbar, 컬러 아이콘 등)과 현재 Milestone Detail 구현의 불일치를 분석합니다.

2-1. 항목별 갭 비교

패턴 요소공통 기준 (다른 탭)Milestone Detail 현재갭
Row 1 (ProjectHeader) 공유 ProjectHeader.tsx (56px), Logo + Project Selector + 6개 Nav 탭 + 알림 + 키보드 + Avatar 자체 breadcrumb 헤더 (~52px). Logo > Project > Milestone. 6개 Nav 탭 없음. 알림/키보드 없음. MISSING ProjectHeader 미사용. 다른 탭 네비게이션 불가.
Row 2 (Subtab Row) 42px, 3~4개 탭, 컬러 아이콘 + Badge count, 우측 Action 버튼 카드 내부에 텍스트 전용 탭 (RESULTS/STATUS/ACTIVITY/ISSUES). 아이콘 없음. Badge 없음. 우측 Action 없음. MISSING 서브탭 Row 패턴 미적용. 카드 안에 내장.
서브탭 아이콘 각 탭마다 컬러 Remix 아이콘 (Indigo/Blue/Green/Red 등) 텍스트만. 아이콘 없음. MISSING

Detail 페이지 패턴 Run Detail: ProjectHeader + 자체 상단 Info + 탭/콘텐츠 자체 헤더 + Info Card + 카드 내 탭 + About 부분 일치 Info Card 패턴은 유사

2-2. 핵심 갭 요약

🔺 총 6개 불일치 항목:
① ProjectHeader.tsx 미사용 → 6개 탭 Nav 접근 불가, 프로젝트 전환 불가
② 서브탭 Row (42px) 미적용 → 카드 내부 텍스트 탭, 공통 패턴 완전 이탈
③ 서브탭 아이콘/Badge 없음 → 시각적 통일성 깨짐
④ Toolbar(Search/Filter/Sort) 없음 → Runs/Issues가 많아지면 탐색 어려움
⑤ STATUS 탭 미구현 → "coming soon" 플레이스홀더 상태
⑥ Milestone Info가 단순 → D-day, TC 통계 요약, 위험도 표시 없음
💡 Detail 페이지 vs List 페이지 차이점:
Milestone Detail은 리스트 페이지가 아닌 Detail 페이지입니다. Run Detail도 ProjectHeader를 사용하지만, 서브탭 Row는 리스트 페이지와 다른 패턴을 씁니다. 따라서 Detail 페이지에 맞는 2-Row Header + Detail Info Bar + Content Tabs 패턴을 적용합니다.
3 표시 정보 적절성 검토

3-1. 있어야 하는데 없는 정보

#누락 정보필요 이유우선순위
1D-day (마감까지 남은 일수)Milestone 관리의 핵심. end_date 대비 오늘 계산. "D-3" 등 즉각적 긴박감 전달.P0
2TC 통계 요약 (Passed/Failed/Blocked/Untested)Info Card에 전체 TC 실행 현황이 없음. 도넛 차트는 Results 탭 안에 숨어있어 한눈에 파악 불가.P0
3Pass Rate (%)Coverage 진행률과 별개로, 실행된 TC 중 Pass 비율이 핵심 지표.P0
4Risk/Warning 인디케이터D-day ≤3d + Pass Rate <70% 같은 위험 상태 시각적 경고.P1
5Assignee 정보 (Run별)Run 카드에 담당자가 표시되지 않음. 누가 어떤 Run을 맡고 있는지 파악 불가.P1
6Status 탭 콘텐츠"Coming soon" 상태. 전체 TC 상태 분포, Sub-Milestone별 진행률 비교 등 필요.P1

4 개선 방향 및 와이어프레임

4-1. 개선 원칙

  • 원칙 1: ProjectHeader.tsx 적용 — 다른 탭 네비게이션을 Milestone Detail에서도 가능하게

E. Status 탭 디자인 (P1, 와이어프레임 수준)

ID요구사항
E-1TC 상태 분포: 수평 스택 바 (Passed green / Failed red / Blocked orange / Retest amber / Untested gray) + 각 상태별 수치 테이블.
E-2Sub-Milestone 비교: 부모 + Sub별 수평 진행률 바 비교. Sub 없으면 숨김.
E-3Risk 배너: "At Risk" (amber bg) 또는 "Critical" (red bg). 조건: D-day≤3d + Pass Rate<70% / D-day≤7d + Pass Rate<50%. 해당 없으면 숨김.
E-4Contributors: 아바타 + 이름 + 실행 TC 수. 기존 About Card Contributors 이동.
