옵션 A: 지역별 분리 ❌

한국/아시아 → Lemon Squeezy

  • GeoIP 기반 분기 필요
  • 복잡도 높음, 유지보수 부담
  • Paddle 승인 전에는 의미 없음

판정: 비추천 — 현 상황에 불필요한 복잡성

Paddle 승인 시 선택적 전환/병행

  • 단일 결제 플로우 → 단순한 구현
  • LS 웹훅 1개만 구현하면 됨
  • Paddle 승인 후 복귀도 용이

4.1 새 파일: src/lib/lemonsqueezy.ts

Paddle의 paddle.ts와 동일한 패턴으로 Lemon Squeezy 체크아웃 모듈을 생성합니다.

// src/lib/lemonsqueezy.ts // Lemon.js는 CDN 스크립트로 로드 (index.html에 추가) // <script src="https://app.lemonsqueezy.com/js/lemon.js" defer></script> declare global { interface Window { createLemonSqueezy: () => void; LemonSqueezy: { Url: { Open: (url: string) => void }; Setup: (config: { eventHandler: (event: any) => void }) => void; }; } } let _onError: ((message: string) => void) | null = null; let _onSuccess: ((event: any) => void) | null = null; export function registerLSErrorHandler(cb: (msg: string) => void) { _onError = cb; } export function registerLSSuccessHandler(cb: (event: any) => void) { _onSuccess = cb; } // Variant ID 매핑 — CEO가 LS Dashboard에서 제공 export const LS_VARIANT_IDS: Record<string, Record<'monthly' | 'annual', string>> = { Starter: { monthly: 'TODO_VARIANT_ID', annual: 'TODO_VARIANT_ID' }, Professional: { monthly: 'TODO_VARIANT_ID', annual: 'TODO_VARIANT_ID' }, 'Enterprise S': { monthly: 'TODO_VARIANT_ID', annual: 'TODO_VARIANT_ID' }, 'Enterprise M': { monthly: 'TODO_VARIANT_ID', annual: 'TODO_VARIANT_ID' }, }; export function initLemonSqueezy() { if (window.createLemonSqueezy) { window.createLemonSqueezy(); } // 이벤트 핸들러 등록 window.LemonSqueezy?.Setup({ eventHandler: (event: any) => { if (event.event === 'Checkout.Success') { _onSuccess?.(event); } }, }); } export function openLSCheckout( planName: string, billingPeriod: 'monthly' | 'annual', userEmail?: string, userId?: string, ): boolean { const variantId = LS_VARIANT_IDS[planName]?.[billingPeriod]; if (!variantId || variantId === 'TODO_VARIANT_ID') { _onError?.('This plan is not yet available. Please contact support.'); return false; } // 체크아웃 URL 구성 (checkout_data.custom으로 user_id 전달) let url = `https://testably.lemonsqueezy.com/checkout/buy/${variantId}`; const params = new URLSearchParams(); if (userEmail) params.set('checkout[email]', userEmail); if (userId) params.set('checkout[custom][user_id]', userId); if (params.toString()) url += '?' + params.toString(); try { window.LemonSqueezy.Url.Open(url); return true; } catch { _onError?.('Payment system is currently unavailable.'); return false; } }

4.2 새 파일: src/lib/payment.ts (Provider 추상화)

// src/lib/payment.ts — Provider 추상화 레이어 import { openPaddleCheckout, registerPaddleErrorHandler } from './paddle'; import { openLSCheckout, registerLSErrorHandler, initLemonSqueezy } from './lemonsqueezy'; type PaymentProvider = 'paddle' | 'lemonsqueezy'; const PROVIDER: PaymentProvider = (import.meta.env.VITE_PAYMENT_PROVIDER as PaymentProvider) || 'lemonsqueezy'; export function getProvider(): PaymentProvider { return PROVIDER; } export function getProviderDisplayName(): string { return PROVIDER === 'paddle' ? 'Paddle' : 'Lemon Squeezy'; } export function initPayment() { if (PROVIDER === 'lemonsqueezy') initLemonSqueezy(); // Paddle은 lazy-init이므로 별도 초기화 불필요 } export function registerPaymentErrorHandler(cb: (msg: string) => void) { if (PROVIDER === 'paddle') registerPaddleErrorHandler(cb); else registerLSErrorHandler(cb); } export async function openCheckout( planName: string, billingPeriod: 'monthly' | 'annual', opts?: { email?: string; userId?: string }, ): Promise<boolean> { if (PROVIDER === 'paddle') { return openPaddleCheckout(planName, billingPeriod); } return openLSCheckout(planName, billingPeriod, opts?.email, opts?.userId); }

연동 범위: 백엔드 (Webhook Edge Function)

5.1 새 Edge Function: supabase/functions/ls-webhook-handler/index.ts

Lemon Squeezy 웹훅을 수신하여 profiles 테이블의 구독 상태를 자동 업데이트합니다. Paddle에는 없던 핵심 기능입니다.

처리할 웹훅 이벤트

이벤트처리 내용우선순위
subscription_created subscription_tier 업데이트, is_trial=false, subscription_ends_at 설정, payment_provider='lemonsqueezy' 저장 P0
subscription_updated 플랜 변경(업/다운그레이드) 시 tier 업데이트, 갱신일 업데이트 P0
subscription_cancelled subscription_ends_at을 현재 빌링 주기 종료일로 설정 (즉시 다운그레이드 아님) P0
subscription_payment_success subscription_ends_at 갱신 (다음 빌링일로 연장) P1
subscription_payment_failed 결제 실패 로그 기록, 선택적으로 사용자에게 알림 P2
order_created 일회성 결제 처리 (현재 미사용, 향후 확장용) P2

5.2 웹훅 핸들러 핵심 로직

// supabase/functions/ls-webhook-handler/index.ts (핵심 구조) import { createClient } from 'https://esm.sh/@supabase/supabase-js@2'; import { createHmac } from 'node:crypto'; Deno.serve(async (req: Request) => { // 1. HMAC 서명 검증 const signature = req.headers.get('X-Signature'); const body = await req.text(); const secret = Deno.env.get('LS_WEBHOOK_SECRET')!; const hmac = createHmac('sha256', secret).update(body).digest('hex'); if (hmac !== signature) return new Response('Invalid signature', { status: 401 }); const payload = JSON.parse(body); const eventName = payload.meta.event_name; const customData = payload.meta.custom_data; // { user_id: '...' } const userId = customData?.user_id; if (!userId) return new Response('Missing user_id', { status: 400 }); const supabase = createClient( Deno.env.get('SUPABASE_URL')!, Deno.env.get('SUPABASE_SERVICE_ROLE_KEY')!, ); // 2. Variant ID → Tier 매핑 const variantTierMap: Record<string, number> = { 'STARTER_MONTHLY_VARIANT': 2, 'STARTER_ANNUAL_VARIANT': 2, 'PRO_MONTHLY_VARIANT': 3, 'PRO_ANNUAL_VARIANT': 3, 'ENT_S_MONTHLY_VARIANT': 4, 'ENT_S_ANNUAL_VARIANT': 4, 'ENT_M_MONTHLY_VARIANT': 5, 'ENT_M_ANNUAL_VARIANT': 5, }; // 3. 이벤트별 처리 switch (eventName) { case 'subscription_created': case 'subscription_updated': { const variantId = String(payload.data.attributes.variant_id); const tier = variantTierMap[variantId] || 2; const renewsAt = payload.data.attributes.renews_at; await supabase.from('profiles').update({ subscription_tier: tier, is_trial: false, subscription_ends_at: renewsAt, payment_provider: 'lemonsqueezy', ls_subscription_id: String(payload.data.id), }).eq('id', userId); break; } case 'subscription_cancelled': { const endsAt = payload.data.attributes.ends_at; await supabase.from('profiles').update({ subscription_ends_at: endsAt, }).eq('id', userId); break; } case 'subscription_payment_success': { const renewsAt = payload.data.attributes.renews_at; await supabase.from('profiles').update({ subscription_ends_at: renewsAt, }).eq('id', userId); break; } } return new Response(JSON.stringify({ received: true }), { status: 200 }); });

5.3 DB 스키마 변경 (Migration)

profiles 테이블에 2개 컬럼 추가가 필요합니다:

-- supabase/migrations/YYYYMMDD_add_payment_provider.sql ALTER TABLE profiles ADD COLUMN IF NOT EXISTS payment_provider TEXT DEFAULT NULL, ADD COLUMN IF NOT EXISTS ls_subscription_id TEXT DEFAULT NULL; COMMENT ON COLUMN profiles.payment_provider IS 'paddle | lemonsqueezy | null(manual)'; COMMENT ON COLUMN profiles.ls_subscription_id IS 'Lemon Squeezy subscription ID for API calls';
왜 payment_provider 컬럼이 필요한가?
Paddle과 LS가 병행될 때, 어떤 provider로 결제했는지 알아야 구독 관리(취소, 변경) API를 올바른 provider로 호출할 수 있습니다. 또한 향후 Paddle 승인 후 마이그레이션 시 어떤 사용자를 전환해야 하는지 식별 가능합니다.
