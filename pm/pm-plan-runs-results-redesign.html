1-3. 현재 문제점

  1. 서브탭 부족: Active/Completed 2개만 존재. 다른 탭은 3~4개 서브탭 + 컬러 아이콘 패턴 사용.
  2. Focus Mode 진입 경로 부재: Runs 리스트에서 바로 Focus Mode로 진입할 수 없음. ContinueRunPanel은 Dashboard 전용.
  3. Stats 카드 위치: 스크롤 영역 내부에 위치하여 스크롤 시 사라짐. Desi 목업처럼 고정 Stats bar가 더 효과적.
  4. Sort 기능 부재: 검색과 필터만 있고 정렬 버튼이 없음.
  5. Run 상태 불완전: DB에는 5개 상태(new, in_progress, paused, under_review, completed)가 있으나, 서브탭은 Active/Completed 2개로만 분류.
  6. Priority 정보 없음: CEO 확정된 Milestone 기한 기반 Priority가 리스트에 표시되지 않음.
  7. Bulk Actions 없음: 체크박스 선택 후 일괄 작업 불가.

C. Toolbar 통일

Milestones/Discovery Log 탭과 동일한 Toolbar 패턴으로 통일합니다:

  • Search: 🔍 Search runs... (flex-1)
  • Result Filter (신규): All Results / Has Failures / All Passed / Has Blocked 드롭다운. v1.1
  • Milestone Filter: 드롭다운으로 Milestone 필터 (현재 사이드바 대체)
  • Filters: Status, Assignee, Tags, Automated/Manual 복합 필터
  • Sort: Created date, Name, Progress, Priority (기본: Priority desc → Created desc)
💡 사이드바 대신 Milestone Filter: Desi 목업의 사이드바(220px Milestones + Tags)는 기각하되, 해당 필터 기능은 Toolbar의 "Milestone ▾" 드롭다운과 "Filters" 패널로 이관합니다. 이를 통해 다른 탭과의 레이아웃 통일성을 유지하면서 동일한 필터링 기능을 제공합니다.

F. Priority 그룹핑

Active 탭에서 Run 카드들을 Priority 그룹별로 나누어 표시합니다:

  • 그룹 헤더: 🔴 CRITICAL (2) — Priority 색상 + 이름 + 카운트
  • 정렬 순서: Critical → High → Medium → Low. 동일 Priority 내에서는 Milestone due date ASC.
  • Completed 탭: Priority 그룹핑 없이 완료 시간 순 정렬 (최근 완료 우선). Result Filter로 실패/통과/차단 Run 필터링 가능.
  • 기존 Milestone 그룹핑은 Toolbar의 "Group by" 옵션에서 선택 가능 (기본: Priority).
